<h1>第13章：分区表</h1>
<blockquote>
<p>提高大表查询性能的利器</p>
</blockquote>
<h2>13.1 分区表概述</h2>
<h3>13.1.1 什么是分区表？</h3>
<p><strong>定义：</strong></p>
<ul>
<li>将一个大表分成多个物理分区</li>
<li>逻辑上仍是一个表</li>
<li>提高查询性能和管理效率</li>
</ul>
<p><strong>优点：</strong></p>
<ul>
<li>✅ 提高查询性能（分区裁剪）</li>
<li>✅ 便于数据管理（删除旧分区）</li>
<li>✅ 提高并发性能</li>
</ul>
<p><strong>缺点：</strong></p>
<ul>
<li>❌ 增加复杂度</li>
<li>❌ 某些查询可能变慢</li>
</ul>
<hr>
<h2>13.2 分区类型 ⭐⭐⭐⭐</h2>
<h3>13.2.1 RANGE分区（范围分区）</h3>
<p><strong>按范围分区：</strong></p>
<pre><code class="language-sql">-- 按年份分区
CREATE TABLE orders (
    id INT NOT NULL,
    user_id INT NOT NULL,
    amount DECIMAL(10,2),
    create_time DATE NOT NULL
)
PARTITION BY RANGE (YEAR(create_time)) (
    PARTITION p2021 VALUES LESS THAN (2022),
    PARTITION p2022 VALUES LESS THAN (2023),
    PARTITION p2023 VALUES LESS THAN (2024),
    PARTITION p2024 VALUES LESS THAN (2025),
    PARTITION p_max VALUES LESS THAN MAXVALUE
);
<p>-- 按ID范围分区
CREATE TABLE users (
id INT NOT NULL,
name VARCHAR(100),
age INT
)
PARTITION BY RANGE (id) (
PARTITION p0 VALUES LESS THAN (1000000),
PARTITION p1 VALUES LESS THAN (2000000),
PARTITION p2 VALUES LESS THAN (3000000),
PARTITION p_max VALUES LESS THAN MAXVALUE
);
</code></pre></p>
<h3>13.2.2 LIST分区（列表分区）</h3>
<p><strong>按列表分区：</strong></p>
<pre><code class="language-sql">-- 按地区分区
CREATE TABLE users (
    id INT NOT NULL,
    name VARCHAR(100),
    province VARCHAR(50)
)
PARTITION BY LIST COLUMNS(province) (
    PARTITION p_north VALUES IN ('北京', '天津', '河北'),
    PARTITION p_east VALUES IN ('上海', '江苏', '浙江'),
    PARTITION p_south VALUES IN ('广东', '福建', '海南'),
    PARTITION p_west VALUES IN ('四川', '重庆', '云南')
);
</code></pre>
<h3>13.2.3 HASH分区（哈希分区）</h3>
<p><strong>按哈希分区：</strong></p>
<pre><code class="language-sql">-- 按用户ID哈希分区（4个分区）
CREATE TABLE orders (
    id INT NOT NULL,
    user_id INT NOT NULL,
    amount DECIMAL(10,2),
    create_time DATETIME
)
PARTITION BY HASH(user_id)
PARTITIONS 4;
<p>-- 数据分布：
-- user_id % 4 = 0 → p0
-- user_id % 4 = 1 → p1
-- user_id % 4 = 2 → p2
-- user_id % 4 = 3 → p3
</code></pre></p>
<h3>13.2.4 KEY分区</h3>
<p><strong>按KEY分区：</strong></p>
<pre><code class="language-sql">-- 类似HASH分区，但使用MySQL内部哈希函数
CREATE TABLE orders (
    id INT NOT NULL,
    user_id INT NOT NULL,
    amount DECIMAL(10,2)
)
PARTITION BY KEY(user_id)
PARTITIONS 4;
</code></pre>
<hr>
<h2>13.3 分区管理</h2>
<h3>13.3.1 查看分区</h3>
<p><strong>查看分区信息：</strong></p>
<pre><code class="language-sql">-- 查看分区
SELECT 
    PARTITION_NAME,
    PARTITION_EXPRESSION,
    PARTITION_DESCRIPTION,
    TABLE_ROWS
FROM information_schema.PARTITIONS
WHERE TABLE_SCHEMA = 'mydb' AND TABLE_NAME = 'orders';
<p>-- 查看分区定义
SHOW CREATE TABLE orders;
</code></pre></p>
<h3>13.3.2 添加分区</h3>
<p><strong>添加RANGE分区：</strong></p>
<pre><code class="language-sql">-- 添加2025年分区
ALTER TABLE orders
ADD PARTITION (PARTITION p2025 VALUES LESS THAN (2026));
</code></pre>
<p><strong>添加HASH分区：</strong></p>
<pre><code class="language-sql">-- 增加分区数量（从4个增加到8个）
ALTER TABLE orders
ADD PARTITION PARTITIONS 4;
</code></pre>
<h3>13.3.3 删除分区</h3>
<p><strong>删除分区：</strong></p>
<pre><code class="language-sql">-- 删除2021年分区（数据也会被删除）
ALTER TABLE orders
DROP PARTITION p2021;
<p>-- 删除多个分区
ALTER TABLE orders
DROP PARTITION p2021, p2022;
</code></pre></p>
<h3>13.3.4 合并分区</h3>
<p><strong>合并分区：</strong></p>
<pre><code class="language-sql">-- 合并p0和p1为p01
ALTER TABLE orders
REORGANIZE PARTITION p0, p1 INTO (
    PARTITION p01 VALUES LESS THAN (2000000)
);
</code></pre>
<h3>13.3.5 拆分分区</h3>
<p><strong>拆分分区：</strong></p>
<pre><code class="language-sql">-- 拆分p_max为p2025和新的p_max
ALTER TABLE orders
REORGANIZE PARTITION p_max INTO (
    PARTITION p2025 VALUES LESS THAN (2026),
    PARTITION p_max VALUES LESS THAN MAXVALUE
);
</code></pre>
<hr>
<h2>13.4 分区裁剪</h2>
<p><strong>什么是分区裁剪？</strong></p>
<ul>
<li>查询时只扫描相关分区</li>
<li>提高查询性能</li>
</ul>
<p><strong>示例：</strong></p>
<pre><code class="language-sql">-- 查询2023年的订单
SELECT * FROM orders WHERE create_time &gt;= '2023-01-01' AND create_time &lt; '2024-01-01';
<p>-- 只扫描p2023分区
EXPLAIN PARTITIONS SELECT * FROM orders WHERE create_time &gt;= '2023-01-01' AND create_time &lt; '2024-01-01';</p>
<p>-- 输出：
-- partitions: p2023
</code></pre></p>
<hr>
<h2>13.5 分区表最佳实践</h2>
<h3>13.5.1 何时使用分区表</h3>
<p><strong>适用场景：</strong></p>
<ul>
<li>✅ 表非常大（&gt;1000万行）</li>
<li>✅ 按时间查询（日志表、订单表）</li>
<li>✅ 需要定期删除旧数据</li>
<li>✅ 数据有明显的分区特征</li>
</ul>
<p><strong>不适用场景：</strong></p>
<ul>
<li>❌ 表不大（&lt;1000万行）</li>
<li>❌ 查询不涉及分区键</li>
<li>❌ 需要频繁跨分区查询</li>
</ul>
<h3>13.5.2 分区键选择</h3>
<p><strong>原则：</strong></p>
<ol>
<li>选择查询中常用的字段</li>
<li>数据分布均匀</li>
<li>便于管理</li>
</ol>
<p><strong>示例：</strong></p>
<pre><code class="language-sql">-- ✅ 好的分区键：create_time（按时间查询多）
PARTITION BY RANGE (YEAR(create_time))
<p>-- ❌ 不好的分区键：status（数据分布不均）
PARTITION BY LIST (status)
</code></pre></p>
<h3>13.5.3 注意事项</h3>
<p><strong>限制：</strong></p>
<ul>
<li>分区键必须是主键或唯一索引的一部分</li>
<li>最多1024个分区</li>
<li>不支持外键</li>
<li>不支持全文索引</li>
</ul>
<p><strong>示例：</strong></p>
<pre><code class="language-sql">-- ❌ 错误：分区键不是主键的一部分
CREATE TABLE orders (
    id INT PRIMARY KEY,
    create_time DATE
)
PARTITION BY RANGE (YEAR(create_time)) (...);
<p>-- ✅ 正确：分区键是主键的一部分
CREATE TABLE orders (
id INT,
create_time DATE,
PRIMARY KEY (id, create_time)
)
PARTITION BY RANGE (YEAR(create_time)) (...);
</code></pre></p>

<h2>13.6 分区表实战案例</h2>
<p><strong>案例：日志表分区</strong></p>
<pre><code class="language-sql">-- 创建日志表（按月分区）
CREATE TABLE access_logs (
    id BIGINT NOT NULL AUTO_INCREMENT,
    user_id INT,
    url VARCHAR(500),
    ip VARCHAR(50),
    create_time DATETIME NOT NULL,
    PRIMARY KEY (id, create_time)
)
PARTITION BY RANGE (TO_DAYS(create_time)) (
    PARTITION p202401 VALUES LESS THAN (TO_DAYS('2024-02-01')),
    PARTITION p202402 VALUES LESS THAN (TO_DAYS('2024-03-01')),
    PARTITION p202403 VALUES LESS THAN (TO_DAYS('2024-04-01')),
    PARTITION p_max VALUES LESS THAN MAXVALUE
);
<p>-- 每月添加新分区
ALTER TABLE access_logs
REORGANIZE PARTITION p_max INTO (
PARTITION p202404 VALUES LESS THAN (TO_DAYS('2024-05-01')),
PARTITION p_max VALUES LESS THAN MAXVALUE
);</p>
<p>-- 删除旧分区（删除3个月前的数据）
ALTER TABLE access_logs
DROP PARTITION p202401;
</code></pre></p>
