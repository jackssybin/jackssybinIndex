<h2>27.2 MySQL内置诊断工具 ⭐⭐⭐⭐⭐</h2>
<h3>27.2.1 SHOW PROCESSLIST</h3>
<p><strong>查看当前正在执行的查询：</strong></p>
<pre><code class="language-sql">-- 查看进程列表
SHOW PROCESSLIST;
<p>-- 查看完整SQL
SHOW FULL PROCESSLIST;</p>
<p>-- 输出字段：
-- Id：线程ID
-- User：用户
-- Host：客户端地址
-- db：数据库
-- Command：命令类型（Query、Sleep等）
-- Time：执行时间（秒）
-- State：状态
-- Info：SQL语句</p>
<p>-- 示例输出：
-- Id  User  Host           db    Command  Time  State           Info
-- 1   root  localhost:3306 test  Query    10    Sending data    SELECT * FROM users WHERE age &gt; 18
-- 2   root  localhost:3307 test  Sleep    100   NULL            NULL
</code></pre></p>
<p><strong>找出慢查询：</strong></p>
<pre><code class="language-sql">-- 查找执行时间&gt;10秒的查询
SELECT * FROM information_schema.PROCESSLIST 
WHERE TIME &gt; 10 AND COMMAND != 'Sleep'
ORDER BY TIME DESC;
<p>-- 杀死慢查询
KILL 线程ID;
</code></pre></p>
<hr>
<h3>27.2.2 EXPLAIN分析查询 ⭐⭐⭐⭐⭐</h3>
<p><strong>基本用法：</strong></p>
<pre><code class="language-sql">EXPLAIN SELECT * FROM users WHERE age &gt; 18;
<p>-- 输出字段：
-- id：查询序号
-- select_type：查询类型（SIMPLE、PRIMARY、SUBQUERY等）
-- table：表名
-- type：访问类型（ALL、index、range、ref、eq_ref、const、system）
-- possible_keys：可能使用的索引
-- key：实际使用的索引
-- key_len：索引长度
-- ref：索引引用
-- rows：扫描行数
-- Extra：额外信息
</code></pre></p>
<p><strong>type字段（重要）：</strong></p>
<pre><code>性能从好到差：
system &gt; const &gt; eq_ref &gt; ref &gt; range &gt; index &gt; ALL
<ul>
<li>system：表只有一行</li>
<li>const：主键或唯一索引查询</li>
<li>eq_ref：唯一索引扫描</li>
<li>ref：非唯一索引扫描</li>
<li>range：范围扫描（BETWEEN、&gt;、&lt;）</li>
<li>index：全索引扫描</li>
<li>ALL：全表扫描（最差）⚠️
</code></pre></li>
</ul>
<p><strong>示例分析：</strong></p>
<pre><code class="language-sql">-- ❌ 全表扫描
EXPLAIN SELECT * FROM users WHERE age &gt; 18;
-- type: ALL
-- rows: 1000000
-- 问题：没有索引
<p>-- ✅ 索引扫描
CREATE INDEX idx_age ON users(age);
EXPLAIN SELECT * FROM users WHERE age &gt; 18;
-- type: range
-- key: idx_age
-- rows: 100000
</code></pre></p>
<p><strong>EXPLAIN ANALYZE（MySQL 8.0+）：</strong></p>
<pre><code class="language-sql">-- 实际执行并分析
EXPLAIN ANALYZE SELECT * FROM users WHERE age &gt; 18;
<p>-- 输出实际执行时间和行数
</code></pre></p>

<h3>27.2.3 SHOW PROFILE</h3>
<p><strong>开启profiling：</strong></p>
<pre><code class="language-sql">-- 开启
SET profiling = 1;
<p>-- 执行查询
SELECT * FROM users WHERE age &gt; 18;</p>
<p>-- 查看profiles
SHOW PROFILES;</p>
<p>-- 输出：
-- Query_ID  Duration  Query
-- 1         0.5       SELECT * FROM users WHERE age &gt; 18</p>
<p>-- 查看详细信息
SHOW PROFILE FOR QUERY 1;</p>
<p>-- 查看CPU、I/O信息
SHOW PROFILE CPU, BLOCK IO FOR QUERY 1;
</code></pre></p>

<h3>27.2.4 Performance Schema ⭐⭐⭐⭐⭐</h3>
<p><strong>开启Performance Schema：</strong></p>
<pre><code class="language-sql">-- 查看是否开启
SHOW VARIABLES LIKE 'performance_schema';
<p>-- 开启（需要重启）
[mysqld]
performance_schema = ON
</code></pre></p>
<p><strong>查看慢查询：</strong></p>
<pre><code class="language-sql">-- 查看执行时间最长的SQL
SELECT 
    DIGEST_TEXT AS query,
    COUNT_STAR AS exec_count,
    AVG_TIMER_WAIT / 1000000000000 AS avg_time_sec,
    SUM_TIMER_WAIT / 1000000000000 AS total_time_sec
FROM performance_schema.events_statements_summary_by_digest
ORDER BY SUM_TIMER_WAIT DESC
LIMIT 10;
</code></pre>
<p><strong>查看表I/O：</strong></p>
<pre><code class="language-sql">-- 查看表I/O统计
SELECT 
    OBJECT_SCHEMA AS db,
    OBJECT_NAME AS table_name,
    COUNT_READ,
    COUNT_WRITE,
    COUNT_FETCH,
    COUNT_INSERT,
    COUNT_UPDATE,
    COUNT_DELETE
FROM performance_schema.table_io_waits_summary_by_table
WHERE OBJECT_SCHEMA NOT IN ('mysql', 'performance_schema', 'information_schema')
ORDER BY COUNT_READ + COUNT_WRITE DESC
LIMIT 10;
</code></pre>
<p><strong>查看锁等待：</strong></p>
<pre><code class="language-sql">-- 查看当前锁等待
SELECT 
    r.trx_id AS waiting_trx_id,
    r.trx_mysql_thread_id AS waiting_thread,
    r.trx_query AS waiting_query,
    b.trx_id AS blocking_trx_id,
    b.trx_mysql_thread_id AS blocking_thread,
    b.trx_query AS blocking_query
FROM information_schema.INNODB_LOCK_WAITS w
INNER JOIN information_schema.INNODB_TRX b ON b.trx_id = w.blocking_trx_id
INNER JOIN information_schema.INNODB_TRX r ON r.trx_id = w.requesting_trx_id;
</code></pre>

<h3>27.3.2 pt-query-digest（慢查询分析）</h3>
<p><strong>分析慢查询日志：</strong></p>
<pre><code class="language-bash"># 分析慢查询日志
pt-query-digest /var/lib/mysql/slow.log
<h1>输出到文件</h1>
<p>pt-query-digest /var/lib/mysql/slow.log &gt; slow_report.txt</p>
<h1>只分析指定时间范围</h1>
<p>pt-query-digest --since '2024-11-10 00:00:00' <br>
--until '2024-11-10 23:59:59' <br>
/var/lib/mysql/slow.log</p>
<h1>只显示前10条</h1>
<p>pt-query-digest --limit 10 /var/lib/mysql/slow.log</p>
<h1>分析binlog</h1>
<p>pt-query-digest --type binlog /var/lib/mysql/mysql-bin.000001
</code></pre></p>
<p><strong>输出解读：</strong></p>
<pre><code># Query 1: 100 QPS, 0.5s avg, ID 0x1234...
# Attribute    pct   total     min     max     avg     95%  stddev  median
# ============ === ======= ======= ======= ======= ======= ======= =======
# Count         10    1000
# Exec time     50     500s    0.1s    2.0s    0.5s    1.0s    0.2s    0.4s
# Lock time      5      5s     0ms    50ms     5ms    10ms     5ms     3ms
# Rows sent      8   10000      10     100      10      20       5      10
# Rows examine  90  900000     900    9000     900    1800     300     800
<p>SELECT * FROM users WHERE age &gt; 18\G
</code></pre></p>
<h3>27.3.3 pt-online-schema-change（在线DDL）</h3>
<pre><code class="language-bash"># 在线修改表结构（不锁表）
pt-online-schema-change \
  --alter &quot;ADD COLUMN email VARCHAR(100)&quot; \
  --execute \
  D=mydb,t=users
<h1>原理：</h1>
<h1>1. 创建新表</h1>
<h1>2. 复制数据</h1>
<h1>3. 创建触发器同步增量数据</h1>
<h1>4. 重命名表</h1>
<p></code></pre></p>
<h3>27.3.4 pt-table-checksum（数据一致性检查）</h3>
<pre><code class="language-bash"># 检查主从数据一致性
pt-table-checksum \
  --host=master_host \
  --user=root \
  --password=password \
  --databases=mydb
<h1>输出：</h1>
<h1>TS            ERRORS  DIFFS  ROWS  CHUNKS  SKIPPED  TIME  TABLE</h1>
<h1>11-10T10:00:00      0      0  1000       1        0   0.5  mydb.users</h1>
<h1>11-10T10:00:01      0      5  2000       1        0   1.0  mydb.orders</h1>
<p></code></pre></p>
<h3>27.3.5 pt-table-sync（数据同步）</h3>
<pre><code class="language-bash"># 同步主从数据
pt-table-sync \
  --execute \
  --sync-to-master \
  h=slave_host,D=mydb,t=orders
</code></pre>
