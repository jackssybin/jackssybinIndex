<h2>附录B：常见问题 FAQ</h2>
<h3>B.1 基础问题</h3>
<p><strong>Q1: Netty 和 NIO 的关系？</strong>
A: Netty 是基于 Java NIO 的网络框架，封装了 NIO 的复杂性，提供更易用的 API。</p>
<p><strong>Q2: 为什么选择 Netty？</strong>
A: 高性能、易用、稳定、社区活跃、被广泛应用于生产环境。</p>
<p><strong>Q3: Netty 适合什么场景？</strong>
A: RPC 框架、IM 系统、游戏服务器、网关、消息中间件等高性能网络应用。</p>
<h3>B.2 性能问题</h3>
<p><strong>Q4: 如何提升 Netty 性能？</strong>
A:</p>
<ul>
<li>使用池化 ByteBuf</li>
<li>使用直接内存</li>
<li>禁用 Nagle 算法</li>
<li>增大缓冲区</li>
<li>使用零拷贝</li>
<li>业务逻辑异步化</li>
</ul>
<p><strong>Q5: 如何避免内存泄漏？</strong>
A:</p>
<ul>
<li>使用 SimpleChannelInboundHandler</li>
<li>手动释放 ByteBuf</li>
<li>启用内存泄漏检测</li>
</ul>
<h3>B.3 开发问题</h3>
<p><strong>Q6: 如何解决粘包/拆包？</strong>
A: 使用 LengthFieldBasedFrameDecoder 或自定义协议。</p>
<p><strong>Q7: 如何实现心跳检测？</strong>
A: 使用 IdleStateHandler 或自定义心跳机制。</p>
<p><strong>Q8: 如何集成 Spring Boot？</strong>
A: 创建 @Component 组件，在 @PostConstruct 中启动服务器。</p>
