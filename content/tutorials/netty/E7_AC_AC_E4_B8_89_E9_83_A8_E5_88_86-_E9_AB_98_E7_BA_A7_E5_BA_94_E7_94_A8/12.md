<h1>第12章：心跳检测与空闲连接管理</h1>
<h2>本章导读</h2>
<p>心跳检测是保持长连接稳定的重要机制。本章将讲解为什么需要心跳、IdleStateHandler 的使用、自定义心跳实现和断线重连机制。</p>
<hr>
<h2>12.1 为什么需要心跳机制</h2>
<h3>12.1.1 问题场景</h3>
<p><strong>问题1：假死连接</strong></p>
<ul>
<li>网络异常导致连接断开</li>
<li>但应用层不知道，仍然保持连接</li>
<li>浪费服务器资源</li>
</ul>
<p><strong>问题2：NAT超时</strong></p>
<ul>
<li>NAT设备会清除长时间空闲的连接</li>
<li>导致连接断开</li>
</ul>
<p><strong>问题3：防火墙</strong></p>
<ul>
<li>防火墙可能关闭长时间空闲的连接</li>
</ul>
<h3>12.1.2 解决方案</h3>
<p><strong>心跳机制</strong>：</p>
<ul>
<li>定期发送心跳包</li>
<li>检测连接是否存活</li>
<li>及时清理死连接</li>
</ul>
<hr>

<pre><code class="language-java">public class CustomHeartbeatHandler extends ChannelInboundHandlerAdapter {
<pre><code>private static final int HEARTBEAT_INTERVAL = 30;  // 心跳间隔（秒）

<p>@Override

<pre><code>// 启动心跳任务
heartbeatFuture = ctx.executor().scheduleAtFixedRate(() -&amp;amp;gt; {
    long currentTime = System.currentTimeMillis();
    long timeSinceLastHeartbeat = currentTime - lastHeartbeatTime;
    
    if (timeSinceLastHeartbeat &amp;amp;gt; HEARTBEAT_TIMEOUT * 1000) {
        // 超时，关闭连接
        System.out.println(&amp;amp;quot;心跳超时，关闭连接&amp;amp;quot;);
        ctx.close();

        System.out.println(&amp;amp;quot;发送心跳&amp;amp;quot;);

</code></pre>

<h2>12.5 连接管理最佳实践</h2>
<pre><code class="language-java">public class ConnectionManager {
<pre><code>private final Map&amp;lt;String, Channel&amp;gt; connections = new ConcurrentHashMap&amp;lt;&amp;gt;();

<p>public void addConnection(String id, Channel channel) {
connections.put(id, channel);</p>
<pre><code>// 监听连接关闭
channel.closeFuture().addListener(future -&amp;amp;gt; {
    connections.remove(id);
    System.out.println(&amp;amp;quot;连接关闭: &amp;amp;quot; + id);
});
</code></pre>
<p>}</p>
<p>public void removeConnection(String id) {
Channel channel = connections.remove(id);
if (channel != null &amp;amp;&amp;amp; channel.isActive()) {
channel.close();
}
}</p>
<p>public void broadcast(String message) {
connections.values().forEach(channel -&amp;gt; {
if (channel.isActive()) {
channel.writeAndFlush(message);
}
});
}</p>
<p>public void startHealthCheck() {
scheduler.scheduleAtFixedRate(() -&amp;gt; {

<h2>12.6 本章小结</h2>
<p>✅ <strong>心跳机制</strong>：保持长连接稳定<br>
✅ <strong>IdleStateHandler</strong>：内置空闲检测<br>
✅ <strong>自定义心跳</strong>：灵活控制<br>
✅ <strong>断线重连</strong>：提高可用性<br>
✅ <strong>连接管理</strong>：统一管理所有连接</p>
<h3>关键要点</h3>
<ol>
<li>心跳间隔建议 30-60 秒</li>
<li>心跳超时建议 3 倍心跳间隔</li>
<li>使用 IdleStateHandler 简化开发</li>
<li>实现断线重连提高可用性</li>
<li>定期清理无效连接</li>
</ol>
