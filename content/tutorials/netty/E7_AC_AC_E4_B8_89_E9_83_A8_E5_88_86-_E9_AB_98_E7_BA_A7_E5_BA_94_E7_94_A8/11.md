<h1>第11章：零拷贝与高性能优化</h1>
<h2>本章导读</h2>
<p>零拷贝是 Netty 高性能的关键技术之一。本章将深入讲解零拷贝的原理、Netty 中的零拷贝实现、FileRegion 文件传输，以及各种性能优化技巧。</p>
<hr>
<h2>11.1 零拷贝技术原理</h2>
<h3>11.1.1 传统数据拷贝</h3>
<p><strong>传统方式（4次拷贝）</strong>：</p>
<pre><code>应用程序读取文件并发送到网络：
<ol>
<li>DMA拷贝：磁盘 → 内核缓冲区</li>
<li>CPU拷贝：内核缓冲区 → 用户缓冲区（应用程序）</li>
<li>CPU拷贝：用户缓冲区 → Socket缓冲区</li>
<li>DMA拷贝：Socket缓冲区 → 网卡</li>
</ol>
<p>总计：4次拷贝，2次CPU拷贝，2次DMA拷贝
</code></pre></p>
<p><strong>问题</strong>：</p>
<ul>
<li>CPU 参与拷贝，占用 CPU 资源</li>
<li>数据在内核态和用户态之间来回拷贝</li>
<li>性能低下</li>
</ul>
<h3>11.1.2 零拷贝方式</h3>
<p><strong>零拷贝（2次拷贝）</strong>：</p>
<pre><code>使用 sendfile 或 transferTo：
<ol>
<li>DMA拷贝：磁盘 → 内核缓冲区</li>
<li>DMA拷贝：内核缓冲区 → 网卡</li>
</ol>
<p>总计：2次拷贝，0次CPU拷贝，2次DMA拷贝
</code></pre></p>
<p><strong>优势</strong>：</p>
<ul>
<li>CPU 不参与拷贝</li>
<li>减少上下文切换</li>
<li>性能提升 2-3 倍</li>
</ul>
<hr>

<h2>11.3 高性能优化技巧</h2>
<h3>11.3.1 使用池化 ByteBuf</h3>
<pre><code class="language-java">// 配置池化分配器
ServerBootstrap bootstrap = new ServerBootstrap();
bootstrap.option(ChannelOption.ALLOCATOR, PooledByteBufAllocator.DEFAULT);
bootstrap.childOption(ChannelOption.ALLOCATOR, PooledByteBufAllocator.DEFAULT);
<p>// 性能提升：减少GC，提高内存分配效率
</code></pre></p>
<h3>11.3.2 使用直接内存</h3>
<pre><code class="language-java">// 使用直接内存（堆外内存）
ByteBuf directBuf = PooledByteBufAllocator.DEFAULT.directBuffer(1024);
<p>// 优势：
// 1. 减少一次内存拷贝（不需要从堆内存拷贝到堆外内存）
// 2. 适合网络I/O
// 3. 不受GC影响
</code></pre></p>
<h3>11.3.3 禁用 Nagle 算法</h3>
<pre><code class="language-java">// 禁用Nagle算法，减少延迟
bootstrap.childOption(ChannelOption.TCP_NODELAY, true);
<p>// Nagle算法：合并小包，减少网络传输次数
// 禁用后：立即发送，适合对延迟敏感的场景
</code></pre></p>
<h3>11.3.4 调整接收/发送缓冲区</h3>
<pre><code class="language-java">// 增大接收缓冲区
bootstrap.childOption(ChannelOption.SO_RCVBUF, 128 * 1024);  // 128KB
<p>// 增大发送缓冲区
bootstrap.childOption(ChannelOption.SO_SNDBUF, 128 * 1024);  // 128KB</p>
<p>// 优势：减少系统调用次数，提高吞吐量
</code></pre></p>
<h3>11.3.5 使用自适应接收缓冲区</h3>
<pre><code class="language-java">// 自适应调整接收缓冲区大小
bootstrap.childOption(ChannelOption.RCVBUF_ALLOCATOR, 
    new AdaptiveRecvByteBufAllocator(
        64,      // 最小值
        2048,    // 初始值
        65536    // 最大值
    )
);
<p>// 优势：根据实际接收数据量动态调整，避免浪费
</code></pre></p>
<h3>11.3.6 设置写缓冲区水位线</h3>
<pre><code class="language-java">// 设置写缓冲区水位线
bootstrap.childOption(ChannelOption.WRITE_BUFFER_WATER_MARK, 
    new WriteBufferWaterMark(
        32 * 1024,   // 低水位线：32KB
        64 * 1024    // 高水位线：64KB
    )
);
<p>// 当写缓冲区超过高水位线时，Channel变为不可写
// 低于低水位线时，Channel变为可写
</code></pre></p>
<h3>11.3.7 批量写入</h3>
<pre><code class="language-java">public class BatchWriteHandler extends ChannelInboundHandlerAdapter {
<pre><code>@Override
public void channelRead(ChannelHandlerContext ctx, Object msg) {
    // 批量写入（不立即刷新）
    for (int i = 0; i &amp;lt; 100; i++) {
        ctx.write(msg);  // 只写入，不刷新
    }

</code></pre>
<p>}

<h2>11.6 本章小结</h2>
<p>本章我们学习了零拷贝和高性能优化：</p>
<p>✅ <strong>零拷贝原理</strong>：减少CPU拷贝，提升性能<br>
✅ <strong>CompositeByteBuf</strong>：组合多个ByteBuf，避免拷贝<br>
✅ <strong>slice/duplicate</strong>：共享数据，零拷贝<br>
✅ <strong>FileRegion</strong>：文件传输零拷贝<br>
✅ <strong>性能优化</strong>：池化、直接内存、TCP_NODELAY等</p>
<h3>关键要点</h3>
<ol>
<li><strong>零拷贝</strong>可以提升 2-3 倍性能</li>
<li><strong>池化 ByteBuf</strong> 减少 GC，提升性能 200%+</li>
<li><strong>直接内存</strong>适合网络 I/O</li>
<li><strong>禁用 Nagle</strong> 算法减少延迟</li>
<li><strong>批量写入</strong>减少系统调用</li>
</ol>
<h3>性能优化清单</h3>
<ul>
<li>✅ 使用 PooledByteBufAllocator</li>
<li>✅ 使用直接内存</li>
<li>✅ 禁用 TCP_NODELAY</li>
<li>✅ 增大接收/发送缓冲区</li>
<li>✅ 使用自适应接收缓冲区</li>
<li>✅ 设置写缓冲区水位线</li>
