<h1>第2章：网络编程基础</h1>
<h2>本章导读</h2>
<p>在深入学习 Netty 之前，我们需要理解网络编程的基础知识。本章将介绍 BIO、NIO、AIO 三种 I/O 模型，深入讲解 Java NIO 的核心组件，分析传统 Socket 编程的痛点，并展示 Netty 如何优雅地解决这些问题。</p>
<hr>
<h2>2.1 BIO、NIO、AIO 概念对比</h2>
<h3>2.1.1 BIO (Blocking I/O) - 同步阻塞 I/O</h3>
<p><strong>定义</strong>：传统的阻塞式 I/O 模型，一个线程处理一个连接。</p>
<p><strong>工作原理</strong>：</p>
<pre><code>客户端连接 → 服务端创建线程 → 线程阻塞等待数据 → 处理数据 → 返回结果
</code></pre>
<p><strong>代码示例</strong>：</p>
<pre><code class="language-java">// 传统BIO服务器
ServerSocket serverSocket = new ServerSocket(8080);
while (true) {
    // accept()会阻塞，直到有客户端连接
    Socket socket = serverSocket.accept();
<pre><code>// 为每个连接创建一个线程
new Thread(() -&amp;gt; {
    try {
        InputStream in = socket.getInputStream();
        byte[] buffer = new byte[1024];
<pre><code>    // read()会阻塞，直到有数据到达
    int len = in.read(buffer);
    
    // 处理数据...
} catch (IOException e) {
    e.printStackTrace();
}
</code></pre>
<p>}).start();
</code></pre></p>
<p>}
</code></pre></p>
<p><strong>特点</strong>：</p>
<table>
<thead>
<tr>
<th>优点</th>
<th>缺点</th>
</tr>
</thead>
<tbody>
<tr>
<td>编程模型简单</td>
<td>一个连接一个线程，资源消耗大</td>
</tr>
<tr>
<td>易于理解和实现</td>
<td>线程上下文切换开销大</td>
</tr>
<tr>
<td></td>
<td>连接数受限于线程数</td>
</tr>
<tr>
<td></td>
<td>大量线程处于阻塞状态，浪费资源</td>
</tr>
</tbody>
</table>
<p><strong>适用场景</strong>：</p>
<ul>
<li>连接数较少且固定的场景</li>
<li>服务器资源充足</li>
<li>对并发要求不高的应用</li>
</ul>

<h3>2.1.2 NIO (Non-blocking I/O) - 同步非阻塞 I/O</h3>
<p><strong>定义</strong>：基于事件驱动的 I/O 模型，一个线程可以处理多个连接。</p>
<p><strong>工作原理</strong>：</p>
<pre><code>多个客户端连接 → Selector轮询 → 检测就绪事件 → 处理就绪的Channel → 继续轮询
</code></pre>
<p><strong>核心组件</strong>：</p>
<ol>
<li><strong>Channel（通道）</strong>：数据传输的管道</li>
<li><strong>Buffer（缓冲区）</strong>：数据的容器</li>
<li><strong>Selector（选择器）</strong>：事件监听和分发</li>
</ol>
<p><strong>代码示例</strong>：</p>
<pre><code class="language-java">// NIO服务器
Selector selector = Selector.open();
ServerSocketChannel serverChannel = ServerSocketChannel.open();
serverChannel.configureBlocking(false);
serverChannel.bind(new InetSocketAddress(8080));
serverChannel.register(selector, SelectionKey.OP_ACCEPT);
<p>while (true) {
// select()会阻塞，但可以监听多个Channel
selector.select();</p>
<pre><code>Iterator&amp;lt;SelectionKey&amp;gt; keys = selector.selectedKeys().iterator();
while (keys.hasNext()) {
    SelectionKey key = keys.next();
    keys.remove();
<pre><code>if (key.isAcceptable()) {
    // 处理连接事件
    ServerSocketChannel server = (ServerSocketChannel) key.channel();
    SocketChannel client = server.accept();
    client.configureBlocking(false);
    client.register(selector, SelectionKey.OP_READ);
} else if (key.isReadable()) {
    // 处理读事件
    SocketChannel client = (SocketChannel) key.channel();
    ByteBuffer buffer = ByteBuffer.allocate(1024);
    client.read(buffer);
    // 处理数据...
}
</code></pre>
<p>}
</code></pre></p>
<p>}
</code></pre></p>
<p><strong>特点</strong>：</p>
<table>
<thead>
<tr>
<th>优点</th>
<th>缺点</th>
</tr>
</thead>
<tbody>
<tr>
<td>一个线程处理多个连接</td>
<td>编程复杂度高</td>
</tr>
<tr>
<td>资源利用率高</td>
<td>需要处理半包、粘包问题</td>
</tr>
<tr>
<td>适合高并发场景</td>
<td>Selector 在 Linux 上基于 epoll，性能好</td>
</tr>
<tr>
<td>减少线程上下文切换</td>
<td>但在 Windows 上性能一般</td>
</tr>
</tbody>
</table>
<p><strong>适用场景</strong>：</p>
<ul>
<li>高并发、连接数多的场景</li>
<li>连接时间长但数据交互少</li>
<li>聊天服务器、推送服务器</li>
</ul>

<h3>2.1.3 AIO (Asynchronous I/O) - 异步非阻塞 I/O</h3>
<p><strong>定义</strong>：真正的异步 I/O，操作完成后会主动通知应用程序。</p>
<p><strong>工作原理</strong>：</p>
<pre><code>发起异步操作 → 立即返回 → 继续其他工作 → 操作完成后回调通知
</code></pre>
<p><strong>代码示例</strong>：</p>
<pre><code class="language-java">// AIO服务器
AsynchronousServerSocketChannel serverChannel = 
    AsynchronousServerSocketChannel.open();
serverChannel.bind(new InetSocketAddress(8080));
<p>serverChannel.accept(null, new CompletionHandler&lt;AsynchronousSocketChannel, Object&gt;() {
@Override
public void completed(AsynchronousSocketChannel client, Object attachment) {
// 继续接受下一个连接
serverChannel.accept(null, this);</p>
<pre><code>    // 处理当前连接
    ByteBuffer buffer = ByteBuffer.allocate(1024);
    client.read(buffer, buffer, new CompletionHandler&amp;lt;Integer, ByteBuffer&amp;gt;() {
        @Override
        public void completed(Integer result, ByteBuffer attachment) {
            // 读取完成，处理数据
            attachment.flip();
            // 处理数据...
        }
<pre><code>    @Override
    public void failed(Throwable exc, ByteBuffer attachment) {
        // 处理失败
    }
});
</code></pre>
<p>}</p>
<p>@Override
public void failed(Throwable exc, Object attachment) {
// 处理失败
}
</code></pre></p>
<p>});
</code></pre></p>

<ul>
<li>理论上适合所有高并发场景</li>

<h2>2.2 Java NIO 核心组件</h2>
<h3>2.2.1 Channel（通道）</h3>
<p><strong>定义</strong>：Channel 是数据传输的管道，类似于流（Stream），但有以下区别：</p>
<table>
<thead>
<tr>
<th>Stream</th>
<th>Channel</th>
</tr>
</thead>
<tbody>
<tr>
<td>单向（输入流或输出流）</td>
<td>双向（可读可写）</td>
</tr>
<tr>
<td>阻塞</td>
<td>可以非阻塞</td>
</tr>
<tr>
<td>直接读写数据</td>
<td>通过 Buffer 读写</td>
</tr>
</tbody>
</table>
<p><strong>主要实现类</strong>：</p>
<pre><code class="language-java">// 文件操作
FileChannel fileChannel = new RandomAccessFile(&quot;file.txt&quot;, &quot;rw&quot;).getChannel();
<p>// 网络操作
ServerSocketChannel serverChannel = ServerSocketChannel.open();  // 服务端
SocketChannel socketChannel = SocketChannel.open();              // 客户端
DatagramChannel datagramChannel = DatagramChannel.open();        // UDP
</code></pre></p>
<p><strong>核心方法</strong>：</p>
<pre><code class="language-java">// 读取数据到Buffer
int bytesRead = channel.read(buffer);
<p>// 从Buffer写入数据
int bytesWritten = channel.write(buffer);</p>
<p>// 设置非阻塞模式
channel.configureBlocking(false);</p>
<p>// 注册到Selector
channel.register(selector, SelectionKey.OP_READ);</p>
<p>// 关闭Channel
channel.close();
</code></pre></p>
<p><strong>示例代码</strong>：</p>
<pre><code class="language-java">public class ChannelDemo {
    public static void main(String[] args) throws IOException {
        // 创建ServerSocketChannel
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.bind(new InetSocketAddress(8080));
        serverChannel.configureBlocking(false);
<pre><code>    System.out.println(&amp;quot;服务器启动，监听端口: 8080&amp;quot;);
<pre><code>while (true) {
    // 非阻塞模式下，accept()立即返回
    SocketChannel clientChannel = serverChannel.accept();
    
    if (clientChannel != null) {
        System.out.println(&amp;amp;quot;客户端连接: &amp;amp;quot; + clientChannel.getRemoteAddress());

        int bytesRead = clientChannel.read(buffer);
        
        if (bytesRead &amp;amp;gt; 0) {

            byte[] data = new byte[buffer.remaining()];
            buffer.get(data);
            System.out.println(&amp;amp;quot;接收到: &amp;amp;quot; + new String(data));
        }
        
        clientChannel.close();

    // 避免CPU空转
    Thread.sleep(1000);
}
</code></pre>
<p>}
</code></pre></p>
<p>}
</code></pre></p>

<h3>2.2.2 Buffer（缓冲区）</h3>
<p><strong>定义</strong>：Buffer 是一个数据容器，本质上是一个数组，用于存储不同类型的数据。</p>
<p><strong>Buffer 类型</strong>：</p>
<pre><code class="language-java">ByteBuffer      // 最常用
CharBuffer
ShortBuffer
IntBuffer
LongBuffer
FloatBuffer
DoubleBuffer
</code></pre>
<p><strong>核心属性</strong>：</p>
<pre><code class="language-java">capacity    // 容量：Buffer的最大数据容量
position    // 位置：下一个要读/写的元素索引
limit       // 限制：第一个不应该读/写的元素索引
mark        // 标记：用于记录当前position的位置
</code></pre>
<p><strong>属性关系</strong>：</p>
<pre><code>0 &lt;= mark &lt;= position &lt;= limit &lt;= capacity
</code></pre>
<p><strong>核心方法</strong>：</p>
<pre><code class="language-java">// 创建Buffer
ByteBuffer buffer = ByteBuffer.allocate(1024);           // 堆缓冲
ByteBuffer directBuffer = ByteBuffer.allocateDirect(1024); // 直接缓冲
<p>// 写入数据
buffer.put((byte) 1);
buffer.put(&quot;Hello&quot;.getBytes());</p>
<p>// 切换到读模式
buffer.flip();  // limit = position; position = 0;</p>
<p>// 读取数据
byte b = buffer.get();
buffer.get(bytes);</p>
<p>// 清空Buffer
buffer.clear();  // position = 0; limit = capacity;</p>
<p>// 压缩Buffer（保留未读数据）
buffer.compact();</p>
<p>// 重置position到mark
buffer.mark();
buffer.reset();</p>
<p>// 倒带（重新读取）
buffer.rewind();  // position = 0;
</code></pre></p>
<p><strong>Buffer 使用流程</strong>：</p>
<pre><code class="language-java">// 1. 创建Buffer
ByteBuffer buffer = ByteBuffer.allocate(1024);
<p>// 2. 写入数据到Buffer
channel.read(buffer);
// 或
buffer.put(&quot;Hello&quot;.getBytes());</p>
<p>// 3. 切换到读模式
buffer.flip();</p>
<p>// 4. 从Buffer读取数据
while (buffer.hasRemaining()) {
byte b = buffer.get();
System.out.print((char) b);
}</p>
<p>// 5. 清空Buffer
buffer.clear();
</code></pre></p>
<p><strong>示例代码</strong>：</p>
<pre><code class="language-java">public class BufferDemo {
    public static void main(String[] args) {
        // 创建Buffer
        ByteBuffer buffer = ByteBuffer.allocate(10);
        System.out.println(&quot;初始状态: &quot; + bufferStatus(buffer));
<pre><code>    // 写入数据
    buffer.put(&amp;quot;Hello&amp;quot;.getBytes());
    System.out.println(&amp;quot;写入后: &amp;quot; + bufferStatus(buffer));
<pre><code>// 切换到读模式

}
System.out.println();

</code></pre>
<p>}</p>

</code></pre></p>
<p>}
</code></pre></p>
<p><strong>输出</strong>：</p>
<pre><code>初始状态: position=0, limit=10, capacity=10

<h3>2.2.3 Selector（选择器）</h3>
<p><strong>定义</strong>：Selector 是 NIO 的核心组件，用于监听多个 Channel 的事件（如连接、读、写）。</p>
<p><strong>核心概念</strong>：</p>
<ul>
<li>一个 Selector 可以监听多个 Channel</li>
<li>一个线程使用一个 Selector 就可以处理多个 Channel</li>
<li>避免了多线程的上下文切换开销</li>
</ul>
<p><strong>事件类型</strong>：</p>
<pre><code class="language-java">SelectionKey.OP_ACCEPT   // 接受连接事件（ServerSocketChannel）
SelectionKey.OP_CONNECT  // 连接就绪事件（SocketChannel）
SelectionKey.OP_READ     // 读就绪事件
SelectionKey.OP_WRITE    // 写就绪事件
</code></pre>
<p><strong>使用流程</strong>：</p>
<pre><code class="language-java">// 1. 创建Selector
Selector selector = Selector.open();
<p>// 2. 创建Channel并设置为非阻塞
ServerSocketChannel serverChannel = ServerSocketChannel.open();
serverChannel.configureBlocking(false);
serverChannel.bind(new InetSocketAddress(8080));</p>
<p>// 3. 注册Channel到Selector
SelectionKey key = serverChannel.register(selector, SelectionKey.OP_ACCEPT);</p>
<p>// 4. 轮询事件
while (true) {
// 阻塞直到有事件发生
int readyChannels = selector.select();</p>
<pre><code>if (readyChannels == 0) {
    continue;
}
<p>// 5. 获取就绪的事件
Set&amp;lt;SelectionKey&amp;gt; selectedKeys = selector.selectedKeys();
Iterator&amp;lt;SelectionKey&amp;gt; keyIterator = selectedKeys.iterator();</p>
<p>while (keyIterator.hasNext()) {
SelectionKey selectionKey = keyIterator.next();</p>
<pre><code>// 6. 处理事件
if (selectionKey.isAcceptable()) {
    // 处理连接事件
} else if (selectionKey.isReadable()) {
    // 处理读事件
} else if (selectionKey.isWritable()) {
    // 处理写事件
}

// 7. 移除已处理的事件
keyIterator.remove();
</code></pre>
<p>}
</code></pre></p>
<p>}
</code></pre></p>
<p><strong>完整示例</strong>：</p>
<pre><code class="language-java">public class SelectorDemo {
    public static void main(String[] args) throws IOException {
        // 创建Selector
        Selector selector = Selector.open();
<pre><code>    // 创建ServerSocketChannel
    ServerSocketChannel serverChannel = ServerSocketChannel.open();
    serverChannel.bind(new InetSocketAddress(8080));
    serverChannel.configureBlocking(false);
<pre><code>// 注册ACCEPT事件
serverChannel.register(selector, SelectionKey.OP_ACCEPT);
System.out.println(&amp;amp;quot;服务器启动，监听端口: 8080&amp;amp;quot;);

while (true) {
    // 阻塞等待事件
    selector.select();
    
    // 获取就绪事件
    Iterator&amp;amp;lt;SelectionKey&amp;amp;gt; keys = selector.selectedKeys().iterator();
    
    while (keys.hasNext()) {
        SelectionKey key = keys.next();
        keys.remove();
        
        if (key.isAcceptable()) {
            // 处理连接事件
            ServerSocketChannel server = (ServerSocketChannel) key.channel();
            SocketChannel client = server.accept();
            client.configureBlocking(false);
            
            // 注册READ事件
            client.register(selector, SelectionKey.OP_READ);
            System.out.println(&amp;amp;quot;客户端连接: &amp;amp;quot; + client.getRemoteAddress());
            
        } else if (key.isReadable()) {
            // 处理读事件
            SocketChannel client = (SocketChannel) key.channel();
            ByteBuffer buffer = ByteBuffer.allocate(1024);
            
            int bytesRead = client.read(buffer);
            if (bytesRead &amp;amp;gt; 0) {
                buffer.flip();
                byte[] data = new byte[buffer.remaining()];
                buffer.get(data);
                System.out.println(&amp;amp;quot;接收到: &amp;amp;quot; + new String(data));
                
                // 回显数据
                buffer.rewind();
                client.write(buffer);
            } else if (bytesRead &amp;amp;lt; 0) {
                // 客户端断开连接
                System.out.println(&amp;amp;quot;客户端断开: &amp;amp;quot; + client.getRemoteAddress());
                key.cancel();
                client.close();
            }
        }
    }
}
</code></pre>
<p>}
</code></pre></p>
<p>}
</code></pre></p>

<h2>2.3 传统 Socket 编程的痛点</h2>
<h3>2.3.1 BIO 的问题</h3>
<p><strong>1. 资源消耗大</strong></p>
<pre><code class="language-java">// 每个连接一个线程
while (true) {
    Socket socket = serverSocket.accept();
    new Thread(() -&gt; {
        // 处理连接...
    }).start();
}
</code></pre>
<p><strong>问题</strong>：</p>
<ul>
<li>1000个连接 = 1000个线程</li>
<li>每个线程占用 1MB 栈空间</li>
<li>大量线程导致频繁的上下文切换</li>
</ul>
<p><strong>2. 阻塞导致资源浪费</strong></p>
<pre><code class="language-java">// read()会阻塞
int len = inputStream.read(buffer);
</code></pre>
<p><strong>问题</strong>：</p>
<ul>
<li>线程大部分时间在等待</li>
<li>CPU 利用率低</li>
<li>无法充分利用系统资源</li>
</ul>
<p><strong>3. 扩展性差</strong></p>
<pre><code class="language-java">// C10K问题：无法支持10000个并发连接
// 原因：线程数量限制
</code></pre>
<h3>2.3.2 NIO 的问题</h3>
<p><strong>1. 编程复杂</strong></p>
<pre><code class="language-java">// 需要手动管理Buffer
buffer.flip();
buffer.clear();
buffer.compact();
<p>// 需要处理半包、粘包
// 需要自己实现协议解析
</code></pre></p>
<p><strong>2. Selector 空轮询 Bug</strong></p>
<pre><code class="language-java">// JDK的epoll空轮询Bug
// 导致CPU 100%占用
selector.select();  // 可能立即返回，而不是阻塞
</code></pre>
<p><strong>3. 需要处理各种边界情况</strong></p>
<pre><code class="language-java">// 连接断开
// 网络异常
// 半包、粘包
// 内存泄漏
</code></pre>
<h3>2.3.3 代码对比</h3>
<p><strong>传统 BIO 服务器</strong>：</p>
<pre><code class="language-java">public class BIOServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(8080);
        System.out.println(&quot;服务器启动...&quot;);
<pre><code>    while (true) {
        // 阻塞等待连接
        Socket socket = serverSocket.accept();
<pre><code>    // 每个连接创建一个线程
    new Thread(() -&amp;amp;gt; {
        try {
            InputStream in = socket.getInputStream();
            OutputStream out = socket.getOutputStream();
            
            byte[] buffer = new byte[1024];
            int len;
            
            // 阻塞读取数据
            while ((len = in.read(buffer)) != -1) {
                String msg = new String(buffer, 0, len);
                System.out.println(&amp;amp;quot;收到: &amp;amp;quot; + msg);
                
                // 回显
                out.write(buffer, 0, len);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }).start();
}
</code></pre>
<p>}
</code></pre></p>
<p>}
</code></pre></p>
<p><strong>问题</strong>：</p>
<ul>
<li>❌ 每个连接一个线程</li>
<li>❌ 线程阻塞等待数据</li>
<li>❌ 无法支持大量并发连接</li>
</ul>
<p><strong>传统 NIO 服务器</strong>：</p>
<pre><code class="language-java">public class NIOServer {
    public static void main(String[] args) throws IOException {
        Selector selector = Selector.open();
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.bind(new InetSocketAddress(8080));
        serverChannel.configureBlocking(false);
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);
<pre><code>    System.out.println(&amp;quot;服务器启动...&amp;quot;);
<pre><code>while (true) {
    selector.select();
    
    Iterator&amp;amp;lt;SelectionKey&amp;amp;gt; keys = selector.selectedKeys().iterator();
    while (keys.hasNext()) {
        SelectionKey key = keys.next();
        keys.remove();
        
        try {
            if (key.isAcceptable()) {
                ServerSocketChannel server = (ServerSocketChannel) key.channel();
                SocketChannel client = server.accept();
                client.configureBlocking(false);
                client.register(selector, SelectionKey.OP_READ);
            } else if (key.isReadable()) {
                SocketChannel client = (SocketChannel) key.channel();
                ByteBuffer buffer = ByteBuffer.allocate(1024);
                
                int len = client.read(buffer);
                if (len &amp;amp;gt; 0) {
                    buffer.flip();
                    // 需要处理半包、粘包问题
                    // 需要手动管理Buffer
                    client.write(buffer);
                } else if (len &amp;amp;lt; 0) {
                    key.cancel();
                    client.close();
                }
            }
        } catch (IOException e) {
            key.cancel();
            key.channel().close();
        }
    }
}
</code></pre>
<p>}
</code></pre></p>
<p>}
</code></pre></p>
<p><strong>问题</strong>：</p>
<ul>
<li>❌ 代码复杂，难以维护</li>
<li>❌ 需要处理各种边界情况</li>
<li>❌ 需要手动处理半包、粘包</li>
<li>❌ 容易出现内存泄漏</li>
</ul>

<h2>2.4 Netty 如何解决这些问题</h2>
<h3>2.4.1 简化 NIO 编程</h3>
<p><strong>Netty 服务器</strong>：</p>
<pre><code class="language-java">public class NettyServer {
    public static void main(String[] args) throws InterruptedException {
        EventLoopGroup bossGroup = new NioEventLoopGroup(1);
        EventLoopGroup workerGroup = new NioEventLoopGroup();
<pre><code>    try {
        ServerBootstrap bootstrap = new ServerBootstrap();
        bootstrap.group(bossGroup, workerGroup)
                .channel(NioServerSocketChannel.class)
                .childHandler(new ChannelInitializer&amp;lt;SocketChannel&amp;gt;() {
                    @Override
                    protected void initChannel(SocketChannel ch) {
                        ch.pipeline().addLast(new EchoHandler());
                    }
                });
<pre><code>    ChannelFuture future = bootstrap.bind(8080).sync();
    System.out.println(&amp;amp;quot;服务器启动...&amp;amp;quot;);
    future.channel().closeFuture().sync();
} finally {
    bossGroup.shutdownGracefully();
    workerGroup.shutdownGracefully();
}
</code></pre>
<p>}
</code></pre></p>
<p>}</p>
<p>class EchoHandler extends ChannelInboundHandlerAdapter {
@Override
public void channelRead(ChannelHandlerContext ctx, Object msg) {
// Netty自动处理Buffer
ctx.write(msg);
}</p>
<pre><code>@Override
public void channelReadComplete(ChannelHandlerContext ctx) {
    ctx.flush();
}
</code></pre>
<p>}
</code></pre></p>
<p><strong>优势</strong>：</p>
<ul>
<li>✅ 代码简洁，易于理解</li>
<li>✅ 自动管理 Buffer</li>
<li>✅ 内置线程模型</li>
<li>✅ 异常处理完善</li>
</ul>
<h3>2.4.2 Netty 的核心解决方案</h3>
<p><strong>1. 优雅的线程模型</strong></p>
<pre><code>Boss线程组（1个线程）
    ↓ 接收连接
Worker线程组（多个线程）
    ↓ 处理I/O
业务线程池（可选）
    ↓ 处理业务逻辑
</code></pre>
<p><strong>2. 强大的 ByteBuf</strong></p>
<pre><code class="language-java">// 自动扩容
ByteBuf buf = Unpooled.buffer();
buf.writeBytes(&quot;Hello&quot;.getBytes());
buf.writeBytes(&quot; World&quot;.getBytes());
<p>// 引用计数，自动释放
buf.release();</p>
<p>// 零拷贝
CompositeByteBuf composite = Unpooled.compositeBuffer();
composite.addComponents(buf1, buf2);
</code></pre></p>
<p><strong>3. 丰富的编解码器</strong></p>
<pre><code class="language-java">// 解决粘包/拆包
pipeline.addLast(new LineBasedFrameDecoder(1024));
pipeline.addLast(new StringDecoder());
<p>// 自定义协议
pipeline.addLast(new LengthFieldBasedFrameDecoder(...));
</code></pre></p>
<p><strong>4. 解决 Selector 空轮询 Bug</strong></p>
<pre><code class="language-java">// Netty内部检测并重建Selector
if (空轮询次数 &gt; 512) {
    重建Selector;
    迁移所有Channel;
}
</code></pre>
<p><strong>5. 内存池技术</strong></p>
<pre><code class="language-java">// 池化ByteBuf，减少GC
PooledByteBufAllocator allocator = PooledByteBufAllocator.DEFAULT;
ByteBuf buf = allocator.buffer();
</code></pre>
