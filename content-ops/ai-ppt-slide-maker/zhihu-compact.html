三大痛点，传统 AI PPT 工具真的解决不了

我用过不下 7 款 AI PPT 工具，它们都有这几个问题：

1. 编造数据是常态

你给它一篇论文，它把你的核心数据改得面目全非。12% 增长率写成 43%，这种错误你拿到会场上讲，直接社死。为什么会这样？因为它们追求速度，不会真的读完你的材料，就是把文本丢进去抓关键词，LLM 天生就会幻觉。

2. 很多内容改不了

号称「导出 PPTX」，结果打开一看，三分之一页面都是图片，想改一个字都改不了。你最后还是得自己重新做一遍，等于 AI 给你搭了个错架子，你还要拆了重建。

3. 作者自审，没人挑错

生成就是交付，写的人就是审的人，自己的问题自己永远发现不了。版式挤了、对比度不对、数据错了，都得你自己检查，AI 没帮你省这个事。

slide-maker 的解法：多智能体分工，让专业的人做专业的事

slide-maker 不是一个 prompt 包办一切，它是真的把做 PPT 拆成了几个不同角色，每个角色只干一件事，而且刻意让评审和作者分开：

角色职责为什么要分开
内容规划通读材料，规划每页讲什么，保证关键信息不丢先保证故事对，再谈好看
美术设计围绕内容设计每页版式配色，不套模板内容决定形式，不是文字倒进模板
独立评审没参与搭建，专门挑错：数字不对、版式不对直接打回旁观者清，避免作者自我陶醉
素材准备并行生成素材，不参与设计决策分工明确，出图更快

这个分工的结果就是：每个数字都能溯源，每一页都设计过，交付前已经被挑过一遍错。

它还有几个细节做得非常到位：

  • 数据图默认原生 PPT 图表 —— 双击就能改数字，不是图片
  • 公式默认原生数学文本 —— 只有二维排版才回退到图片，能编辑尽量编辑
  • 支持任意比例画布 —— 不止 16:9，小红书 3:4、9:16 竖版、A4 打印都能自动排版
  • 口味可沉淀 —— 你多次调整的偏好会存成文件，后续生成默认继承，越用越顺手

五分钟安装上手攻略

环境依赖（必须先装）

slide-maker 需要三个系统依赖：

系统LibreOffice 安装SVG 栅格化安装
macOSbrew install --cask libreofficebrew install librsvg
Linuxsudo apt install libreofficesudo apt install librsvg2-bin
Windowswinget install TheDocumentFoundation.LibreofficeChrome/Edge 已安装即可

安装脚本

# 克隆安装
git clone --depth 1 https://github.com/addsumtech/slides_maker.git
cd slides_maker
python3 -m pip install -r skills/slide-maker/requirements.txt
python3 skills/slide-maker/scripts/install_skill.py --target both
  • both 表示同时给 Claude Code 和 Codex 安装，只用一个可以换成 claude 或 codex
  • Claude Code 用户也可以直接从插件市场装：/plugin marketplace add addsumtech/slides_maker

使用方式

安装完成后，在 Claude Code 里直接输入：

/slide-maker

然后跟着它的提问一步步回答：模板、受众、时长、语言，每题都有现成选项，方向键选择回车就行。确认完它会先给你看逐页结构，你同意了再开始生成。

赶时间也可以一句话开场：

用 slide-maker 按 paper.pdf 做一份 12 分钟的组会汇报 PPT

谁该用？谁不该用？

我的判断很清晰：

✅ 你应该用 slide-maker，如果：

  • 你做 PPT 是组会汇报/毕业答辩/会议报告 —— 对数据准确性要求高，容不下错
  • 你需要生成后继续打磨 —— 希望拿到可编辑的文件，不是一张截图
  • 你讨厌自己检查错 —— 希望有 AI 先帮你把一遍关
  • 你需要多尺寸复用 —— 同一份内容要出 PPT 也要出小红书图文

❌ 你不该用 slide-maker，如果：

  • 你就是想随便玩玩，十秒出个片交差 —— 它要真读材料真评审，比一拍脑袋的工具慢
  • 你需要非常花哨的动画和网页交互 —— 它聚焦可编辑 PPTX，动画只做原生点击渐显
  • 你需要云端在线分享 —— 它输出本地文件，不搞云端托管

最后一句话总结

AI 工具发展到今天，其实已经分成了两条路线：

  • 速度路线：拼「多快好省」，十秒出十几页，适合低风险场景随便玩玩
  • 可靠路线：像 slide-maker 这样，拼「准确、可编、有人审」，解决真问题

当你这份 PPT 真的要拿去讲、要对你的听众负责的时候，后者显然更重要。

slide-maker 没有发明什么新概念，但它把「不编造、可编辑、要审校」这几件简单朴素的事真的做到位了。这在今天遍地都是「十秒出片」的 AI PPT 市场上，反而成了稀缺品。
