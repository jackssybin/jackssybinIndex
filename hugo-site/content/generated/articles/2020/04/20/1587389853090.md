        <section class="core-intro">
    <h3>重写导读：Linux Dump 文件分析</h3>
    <p>这篇文章是线上排障类内容的核心入口，适合在 Java 进程内存异常、CPU 异常或服务不稳定时作为排查流程参考。</p>
    <ul>
      <li>先保留现场，再分析 Dump、线程和系统资源。</li>
<li>排障时要把 JVM 工具和 Linux 命令结合使用。</li>
<li>适合与 jstack、CPU/内存占用、GC 相关文章联读。</li>
    </ul>
  </section>
        <div id="more" class="vditor-reset post__content"><p>dump文件传输到本地进行分析， 常常需要大量的等待时间。 使用IBM的eclipse的MAT工具可以直接在服务器上进行快速DUMP分析。</p>
<h3>运行环境要求</h3>
<ul>
<li>linux操作系统</li>
<li>JDK8 以上</li>
</ul>
<h3>下载MAT的linux版本</h3>
<p><a href="http://www.eclipse.org/mat/downloads.php">Eclipse的MAT工具下载链接</a>
MAT支持各种操作系统，找到Linux版本下载下来</p>
<pre><code># 运行uname -m 看一下linux是 x86_64还是 x86的帮助你选择下载那个版本。
uname -m
#x86_64

</code></pre>
<pre><code>http://iso.mirrors.ustc.edu.cn/eclipse/mat/1.8/rcp/MemoryAnalyzer-1.8.0.20180604-linux.gtk.x86_64.zip

</code></pre>
<h3>解压配置MAT基本参数</h3>
<pre><code>unzip MemoryAnalyzer-1.8.0.20180604-linux.gtk.x86_64.zip
## 修改MAT的内存大小， 注意这个大小要根据你dump文件大小来的，如果dump文件是5GB那么 这里最好配&gt;5GB 否则会报MAT内存不足的异常
## 修改MemoryAnalyzer.ini 的 -Xmx6024m
vi MemoryAnalyzer.ini

</code></pre>
<h3>jmap dump整个堆</h3>
<pre><code>jmap -dump:format=b,file=jmap.info PID
</code></pre>
<h3>MAT分析 dump</h3>
<pre><code> ./ParseHeapDump.sh jmap.info  org.eclipse.mat.api:suspects org.eclipse.mat.api:overview org.eclipse.mat.api:top_components

</code></pre>
<h3>等待结果….</h3>
<p>结果会生产如下三个zip文件，很小可以直接拷贝到本机</p>
<pre><code>jmap_Leak_Suspects.zip
jmap_System_Overview.zip
jmap_Top_Components.zip

</code></pre>
<h3>查看报告结果</h3>
<p>有两种查看报告的方法</p>
<ul>
<li>直接把zip下载到本地，然后解压用浏览器查看index.html</li>
<li>把zip下载到本地， 用MAT可视化工具解析zip</li>
</ul>
<h4>如果有异常Unable to initialize GTK+</h4>
<p>遇到这个问题的话，<a href="http://xn--ParseHeapDump-ho1ul02hsg5c.sh">是因为ParseHeapDump.sh</a>
里面需要调用GTK的一些东西。解决方法：</p>
<pre><code>vi ParseHeapDump.sh
#注释掉 &quot;$(dirname -- &quot;$0&quot;)&quot;/MemoryAnalyzer -consolelog -application org.eclipse.mat.api.parse &quot;$@&quot;这一行
#然后加入下面
#注意plugins/org.eclipse.equinox.launcher_1.5.0.v20180512-1130.jar要根据你自己本地的文件名做修改调整
java -Xmx4g -Xms4g \
-jar  plugins/org.eclipse.equinox.launcher_1.5.0.v20180512-1130.jar \
-consoleLog -consolelog -application org.eclipse.mat.api.parse &quot;$@&quot;

</code></pre>
<p>然后继续运行</p>
<pre><code> ./ParseHeapDump.sh jmap.info  org.eclipse.mat.api:suspects org.eclipse.mat.api:overview org.eclipse.mat.api:top_components
</code></pre>
