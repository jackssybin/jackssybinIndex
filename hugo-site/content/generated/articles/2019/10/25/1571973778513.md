        <section class="article-guide">
      <div><strong>专题</strong><a href="/topics/linux-ops.html">Linux 运维与部署</a></div>
      <div><strong>导读</strong><span>本文归入「Linux 运维与部署」专题，主要记录：1.根据端口查进程 lsof i:port netstat nap | grep port 2.根据进程号查端口: lsof i|grep pid netstat nap | grep pid 3.根据进程名查找pid、port： ps ef |grep tomcat ps ef |grep port(根据port查找...</span></div>
  </section>
        
        <div id="more" class="vditor-reset post__content"><p>1.根据端口查进程</p>
<pre><code>lsof -i:port
netstat -nap | grep port
</code></pre>
<p>2.根据进程号查端口:</p>
<pre><code>lsof -i|grep pid
netstat -nap | grep pid
</code></pre>
<p>3.根据进程名查找pid、port：</p>
<pre><code>ps -ef |grep tomcat
ps -ef |grep port(根据port查找相关进程)
ps -ef |grep pid(根据pid查找相关进程)
</code></pre>
<p>4.根据进程号查服务路径：</p>
<pre><code>ll /proc/26357/cwd    #26357是进程号
1 root root 0 Oct 25 10:08 /proc/26357/cwd -&gt; /root/data/proxy_pool/Api/
</code></pre>
<p>5.查询所有进程号 top</p>
<pre><code>top
</code></pre>
<p>6.查看进程中的线程号信息</p>
<pre><code>ps -T -p 18043   # ps 语法

1.top -H -p 18043  #top 实时的哈
shift+H开启show threads on功能，展示线程资源占用情况
　　找到消耗CPU等最多的PID为:18045
2.  printf &quot;%x\n&quot; 18045  --&gt;467d
3. jstack 18043|grep 467d  #(定位到线程)
&gt;&gt;&quot;VM Thread&quot; os_prio=0 tid=0x00007f36e406f800 nid=0x467d runnable

</code></pre>
