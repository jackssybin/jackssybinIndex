        <div id="more" class="vditor-reset post__content"><h2>多种方法实现 python 线程池</h2>
<h4>一、 既然多线程可以缩短程序运行时间，那么，是不是线程数量越多越好呢？</h4>
<p>显然，并不是，每一个线程的从生成到消亡也是需要时间和资源的，太多的线程会占用过多的系统资源（内存开销，cpu开销），而且生成太多的线程时间也是可观的，很可能会得不偿失，这里给出一个最佳线程数量的计算方式：
<strong>最佳线程数的获取：</strong></p>
<p>1、通过用户慢慢递增来进行性能压测，观察QPS（即每秒的响应请求数，也即是最大吞吐能力。），响应时间</p>
<p>2、根据公式计算:服务器端最佳线程数量=((线程等待时间+线程cpu时间)/线程cpu时间) * cpu数量</p>
<p>3、单用户压测，查看CPU的消耗，然后直接乘以百分比，再进行压测，一般这个值的附近应该就是最佳线程数量。</p>
<h4>二、如何实现线程池？</h4>
<p>1、使用threadpool模块，这是个python的第三方模块，支持python2和python3</p>
<pre><code>#! /usr/bin/env python
# -*- coding: utf-8 -*-

import threadpool
import time

def sayhello (a):
    print(&quot;hello: &quot;+a)
    time.sleep(2)

def main():
    global result
    seed=[&quot;a&quot;,&quot;b&quot;,&quot;c&quot;]
    start=time.time()
    task_pool=threadpool.ThreadPool(5)
    requests=threadpool.makeRequests(sayhello,seed)
    for req in requests:
        task_pool.putRequest(req)
    task_pool.wait()
    end=time.time()
    time_m = end-start
    print(&quot;time: &quot;+str(time_m))
    start1=time.time()
    for each in seed:
        sayhello(each)
    end1=time.time()
    print(&quot;time1: &quot;+str(end1-start1))

if __name__ == '__main__':
    main()
</code></pre>
<p>threadpool是一个比较老的模块了，现在虽然还有一些人在用，但已经不再是主流了，关于python多线程，现在已经开始步入未来（future模块）了</p>
<p>2、未来：使用concurrent.futures模块，这个模块是python3中自带的模块
#测试代码</p>
<pre><code>from concurrent.futures import ProcessPoolExecutor

def testProcessThread():
    for url in URLS:
        future = executorProcessThread.submit(load_url, url)
        print(future.done())


def testThread():#submit 返回无序
    for url in URLS:

        print(future.done())

def testThreadMap():#按照顺序执行
    executor.map(load_url, URLS)


def testThreadWait():

    # print(wait(f_list, return_when='FIRST_COMPLETED')) #FIRST_COMPLETED参数，程序并不会等到线程池里面所有的任务都完成。
