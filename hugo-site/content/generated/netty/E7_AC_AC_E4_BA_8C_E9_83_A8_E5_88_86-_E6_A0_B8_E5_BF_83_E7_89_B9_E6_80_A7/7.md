          <div class="vditor-reset post__content"><h1>第7章：编解码器（Codec）</h1>
<h2>本章导读</h2>
<p>编解码器是 Netty 处理网络数据的核心组件。本章将讲解为什么需要编解码器、Netty 内置的编解码器、如何自定义编解码器，以及如何解决粘包/拆包问题。</p>
<hr>
<h2>7.1 为什么需要编解码器</h2>
<h3>7.1.1 网络传输的问题</h3>
<p><strong>问题</strong>：网络传输只能传输字节流</p>
<pre><code>应用层对象 → 字节流 → 网络传输 → 字节流 → 应用层对象
          编码                      解码
</code></pre>
<p><strong>示例</strong>：</p>
<pre><code class="language-java">// 发送端：需要将对象转换为字节
User user = new User(&quot;Alice&quot;, 25);
byte[] bytes = serialize(user);  // 编码
channel.writeAndFlush(bytes);
<p>// 接收端：需要将字节转换为对象
byte[] bytes = receive();
User user = deserialize(bytes);  // 解码
</code></pre></p>
<h3>7.1.2 编解码器的作用</h3>
<p><strong>编码器（Encoder）</strong>：</p>
<ul>
<li>将消息对象转换为字节流</li>
<li>出站操作（写入数据时）</li>
</ul>
<p><strong>解码器（Decoder）</strong>：</p>
<ul>
<li>将字节流转换为消息对象</li>
<li>入站操作（读取数据时）</li>
</ul>
<p><strong>编解码器（Codec）</strong>：</p>
<ul>
<li>同时包含编码和解码功能</li>
</ul>
<hr>
<h2>7.2 内置编解码器</h2>
<h3>7.2.1 字符串编解码器</h3>
<pre><code class="language-java">pipeline.addLast(&quot;decoder&quot;, new StringDecoder(CharsetUtil.UTF_8));

<p>// 使用

@Override

System.out.println(&quot;收到字符串: &quot; + msg);
ctx.writeAndFlush(&quot;回复: &quot; + msg);  // 自动编码为字节
}
}
</code></pre></p>
<h3>7.2.2 对象编解码器</h3>
<pre><code class="language-java">// Java 序列化（不推荐，性能差）

pipeline.addLast(new ObjectEncoder());
<p>// 使用
public class ObjectHandler extends SimpleChannelInboundHandler&lt;Serializable&gt; {
@Override

}
</code></pre></p>
<h3>7.2.3 Base64 编解码器</h3>
<pre><code class="language-java">pipeline.addLast(new Base64Decoder());
pipeline.addLast(new Base64Encoder());
</code></pre>
<h3>7.2.4 压缩编解码器</h3>
<pre><code class="language-java">// Gzip 压缩
pipeline.addLast(new JdkZlibEncoder(ZlibWrapper.GZIP));
pipeline.addLast(new JdkZlibDecoder(ZlibWrapper.GZIP));
<p>// Snappy 压缩
pipeline.addLast(new SnappyFrameEncoder());
pipeline.addLast(new SnappyFrameDecoder());
</code></pre></p>

<h2>7.4 解决粘包/拆包问题</h2>
<h3>7.4.1 什么是粘包/拆包</h3>
<p><strong>粘包</strong>：多个数据包粘在一起</p>
<pre><code>发送: [包1] [包2] [包3]
接收: [包1包2包3]
</code></pre>
<p><strong>拆包</strong>：一个数据包被拆分</p>
<pre><code>发送: [大包]
接收: [大包的前半部分] [大包的后半部分]
</code></pre>
<p><strong>原因</strong>：</p>
<ul>
<li>TCP 是流式协议，没有消息边界</li>
<li>Nagle 算法合并小包</li>
<li>MSS 限制导致拆包</li>
</ul>
<h3>7.4.2 解决方案</h3>
<p><strong>方案1：固定长度</strong></p>
<pre><code class="language-java">// 每个消息固定 100 字节
pipeline.addLast(new FixedLengthFrameDecoder(100));
</code></pre>
<p><strong>方案2：分隔符</strong></p>
<pre><code class="language-java">// 使用换行符分隔
pipeline.addLast(new LineBasedFrameDecoder(1024));
<p>// 使用自定义分隔符
ByteBuf delimiter = Unpooled.copiedBuffer(&quot;$$&quot;.getBytes());
pipeline.addLast(new DelimiterBasedFrameDecoder(1024, delimiter));
</code></pre></p>
<p><strong>方案3：长度字段</strong>（推荐）</p>
<pre><code class="language-java">/**
 * 消息格式：
 * +--------+----------+
 * | Length | Content  |
 * | 4 byte | N bytes  |
 * +--------+----------+
 */
pipeline.addLast(new LengthFieldBasedFrameDecoder(
    1024,    // maxFrameLength: 最大帧长度
    0,       // lengthFieldOffset: 长度字段偏移量
    4,       // lengthFieldLength: 长度字段长度
    0,       // lengthAdjustment: 长度调整值
    4        // initialBytesToStrip: 跳过的字节数
));
<p>pipeline.addLast(new LengthFieldPrepender(4));  // 编码器
</code></pre></p>
<p><strong>方案4：自定义协议</strong></p>
<pre><code class="language-java">public class CustomFrameDecoder extends ByteToMessageDecoder {
<pre><code>private static final int HEADER_SIZE = 8;  // 魔数(4) + 长度(4)
private static final int MAGIC_NUMBER = 0x12345678;
<p>@Override
protected void decode(ChannelHandlerContext ctx, ByteBuf in, List&amp;lt;Object&amp;gt; out) {
// 至少需要头部
if (in.readableBytes() &amp;lt; HEADER_SIZE) {
return;
}</p>
<pre><code>in.markReaderIndex();
<p>// 读取魔数
int magic = in.readInt();
if (magic != MAGIC_NUMBER) {
throw new RuntimeException(&amp;amp;quot;非法的魔数: &amp;amp;quot; + magic);
}</p>
<p>// 读取长度
int length = in.readInt();</p>
<p>// 检查数据是否完整
if (in.readableBytes() &amp;amp;lt; length) {
in.resetReaderIndex();
return;
}</p>
<p>// 读取数据
byte[] data = new byte[length];
in.readBytes(data);</p>
<p>out.add(data);
</code></pre></p>
<p>}
</code></pre></p>
<p>}
</code></pre></p>

<h2>7.5 常用编解码器详解</h2>
<h3>7.5.1 LengthFieldBasedFrameDecoder</h3>
<p><strong>参数说明</strong>：</p>
<pre><code class="language-java">new LengthFieldBasedFrameDecoder(
    maxFrameLength,        // 最大帧长度
    lengthFieldOffset,     // 长度字段偏移量
    lengthFieldLength,     // 长度字段长度（1/2/3/4/8字节）
    lengthAdjustment,      // 长度调整值
    initialBytesToStrip    // 跳过的字节数
);
</code></pre>
<p><strong>示例1：长度字段在开头</strong></p>
<pre><code>消息格式：
+--------+----------+
| Length | Content  |
| 2 byte | N bytes  |
+--------+----------+
<p>配置：
new LengthFieldBasedFrameDecoder(1024, 0, 2, 0, 2);
</code></pre></p>
<p><strong>示例2：长度字段在中间</strong></p>
<pre><code>消息格式：
+------+--------+----------+
| Type | Length | Content  |
| 1 B  | 2 B    | N bytes  |
+------+--------+----------+
<p>配置：
new LengthFieldBasedFrameDecoder(1024, 1, 2, 0, 3);
</code></pre></p>
<p><strong>示例3：长度包含头部</strong></p>
<pre><code>消息格式：
+--------+----------+
| Length | Content  |  Length = Content长度 + 4
| 4 B    | N bytes  |
+--------+----------+
<p>配置：
new LengthFieldBasedFrameDecoder(1024, 0, 4, -4, 4);
</code></pre></p>
<h3>7.5.2 完整示例</h3>
<pre><code class="language-java">public class CodecServer {
<pre><code>public static void main(String[] args) throws Exception {

    EventLoopGroup workerGroup = new NioEventLoopGroup();
<pre><code>try {

            .channel(NioServerSocketChannel.class)
            .childHandler(new ChannelInitializer&amp;amp;lt;SocketChannel&amp;amp;gt;() {
                @Override
                protected void initChannel(SocketChannel ch) {
                    ChannelPipeline pipeline = ch.pipeline();
<pre><code>                // 1. 解决粘包/拆包
                pipeline.addLast(new LengthFieldBasedFrameDecoder(
                        1024, 0, 4, 0, 4));
                pipeline.addLast(new LengthFieldPrepender(4));
                
                // 2. 自定义编解码器
                pipeline.addLast(new MessageDecoder());
                pipeline.addLast(new MessageEncoder());
                
                // 3. 业务Handler
                pipeline.addLast(new MessageHandler());
            }
        });

</code></pre>
<p>} finally {

}
</code></pre></p>
<p>}
</code></pre></p>
<p>}</p>

</code></pre></p>
<p>}

<p>}

<h2>7.7 本章小结</h2>
<p>本章我们学习了编解码器：</p>
<p>✅ <strong>编解码器的作用</strong>：对象与字节流的转换<br>
✅ <strong>内置编解码器</strong>：String、Object、Base64、压缩等<br>
✅ <strong>自定义编解码器</strong>：MessageToByteEncoder、ByteToMessageDecoder<br>
✅ <strong>粘包/拆包</strong>：固定长度、分隔符、长度字段、自定义协议<br>
✅ <strong>LengthFieldBasedFrameDecoder</strong>：最常用的解决方案<br>
✅ <strong>Protocol Buffers</strong>：高效的序列化方案</p>
<h3>关键要点</h3>
<ol>
<li><strong>编码器</strong>是出站操作，<strong>解码器</strong>是入站操作</li>
<li><strong>粘包/拆包</strong>是 TCP 的固有问题，需要应用层解决</li>
<li><strong>LengthFieldBasedFrameDecoder</strong> 是最常用的解决方案</li>
<li><strong>Protocol Buffers</strong> 性能优于 Java 序列化</li>
<li>自定义编解码器要处理好<strong>半包</strong>问题</li>
</ol>
