<h2>本章概述</h2>
<p>Spring Boot 4 在依赖注入和配置管理方面带来了多项改进，使得应用配置更加灵活、类型安全，并且启动性能得到显著提升。</p>
<p><strong>本章重点</strong>:</p>
<ul>
<li>✅ 新的 Bean 注册机制</li>
<li>✅ 配置属性绑定增强</li>
<li>✅ 条件注解的改进</li>
<li>✅ 启动性能优化</li>
<li>✅ 与 Spring Boot 3 的对比</li>
</ul>
<h2>3.1 新的 Bean 注册机制</h2>
<h3>3.1.1 编程式 Bean 注册 API 改进</h3>
<p>Spring Boot 4 提供了更简洁、类型安全的 Bean 注册 API。</p>
<h4>Spring Boot 3 方式</h4>
<pre><code class="language-java">@Configuration
public class AppConfig {
<pre><code>@Bean
public DataSource dataSource() {
    HikariDataSource ds = new HikariDataSource();
    ds.setJdbcUrl(&amp;quot;jdbc:mysql://localhost:3306/mydb&amp;quot;);
    ds.setUsername(&amp;quot;root&amp;quot;);
    ds.setPassword(&amp;quot;password&amp;quot;);
    return ds;
}
<p>@Bean
public UserService userService(DataSource dataSource) {
return new UserService(dataSource);
}
</code></pre></p>
<p>}
</code></pre></p>
<h4>Spring Boot 4 新方式</h4>
<pre><code class="language-java">@Configuration
public class AppConfig {
<pre><code>/**
 * 使用函数式 Bean 注册（更好的性能）
 */
@Bean
public DataSource dataSource(DataSourceProperties properties) {
    return DataSourceBuilder
        .create()
        .url(properties.getUrl())
        .username(properties.getUsername())
        .password(properties.getPassword())
        .build();
}
<p>/**</p>
<ul>
<li>使用 Supplier 延迟初始化
*/
@Bean
@Lazy
public Supplier&amp;lt;ExpensiveService&amp;gt; expensiveServiceSupplier() {
return () -&amp;gt; new ExpensiveService();
}
</code></pre></li>
</ul>
<p>}
</code></pre></p>

<h4>2. 动态 Bean 注册器</h4>
<p><strong>PluginRegistrar.java</strong>:</p>
<pre><code class="language-java">package com.example.beans.config;
<p>import com.example.beans.plugin.*;

import org.slf4j.LoggerFactory;

<p>@Override

<pre><code> // 注册所有插件

</code></pre></p>

<p>private void registerPlugin(

<pre><code> RootBeanDefinition beanDefinition = new RootBeanDefinition(pluginClass);

</code></pre></p>
</li>

<h4>4. 服务类</h4>
<p><strong>NotificationService.java</strong>:</p>
<pre><code class="language-java">package com.example.beans.service;
<p>import com.example.beans.plugin.Plugin;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.stereotype.Service;</p>
<p>import java.util.List;

<p>@Service
public class NotificationService {

<pre><code>private final Supplier&amp;lt;Map&amp;lt;String, Plugin&amp;gt;&amp;gt; pluginMapSupplier;
<p>public NotificationService(Supplier&amp;lt;Map&amp;lt;String, Plugin&amp;gt;&amp;gt; pluginMapSupplier) {

}</p>
<p>/**</p>
<ul>
<li>
<p>发送通知到指定渠道

<p>Plugin plugin = plugins.get(channel);

return;
}</p>
<p>if (!plugin.isEnabled()) {

<p>plugin.execute(message);
}</p>
</li>
</ul>
<p>/**</p>
<ul>
<li>
<p>发送通知到所有启用的渠道

<p>plugins.values().stream()
.filter(Plugin::isEnabled)

<a href="http://log.info">log.info</a>(&amp;quot;Broadcasting to {}&amp;quot;, plugin.getName());

}</p>
</li>
</ul>
<p>/**</p>
<ul>
<li>获取所有启用的插件
*/
public List&amp;lt;String&amp;gt; getEnabledPlugins() {
return pluginMapSupplier.get().values().stream()
.filter(Plugin::isEnabled)
.map(Plugin::getName)
.toList();
}
</code></pre></li>
</ul>
<p>}
</code></pre></p>

<h2>3.2 配置属性绑定增强</h2>

<h2>3.3 条件注解的改进</h2>
<h3>3.3.1 新的条件注解</h3>
<p>Spring Boot 4 引入了更灵活的条件注解。</p>
<h4>案例：条件化配置</h4>
<p><strong>ConditionalConfig.java</strong>:</p>
<pre><code class="language-java">package com.example.config;
<p>import org.springframework.boot.autoconfigure.condition.*;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.core.env.Environment;</p>
<p>@Configuration
public class ConditionalConfig {</p>
<pre><code>/**
 * 仅在生产环境启用
 */
@Bean
@ConditionalOnProperty(
    prefix = &amp;quot;app&amp;quot;,
    name = &amp;quot;environment&amp;quot;,
    havingValue = &amp;quot;production&amp;quot;
)
public ProductionService productionService() {
    return new ProductionService();
}
<p>/**</p>
<ul>
<li>仅在开发环境启用
*/
@Bean
@ConditionalOnProperty(
prefix = &amp;quot;app&amp;quot;,
name = &amp;quot;environment&amp;quot;,
havingValue = &amp;quot;development&amp;quot;,
matchIfMissing = true
)
public DevelopmentService developmentService() {
return new DevelopmentService();
}</li>
</ul>
<p>/**</p>
<ul>
<li>Spring Boot 4 - 多条件组合
*/
@Bean
@ConditionalOnProperty(prefix = &amp;quot;app.cache&amp;quot;, name = &amp;quot;enabled&amp;quot;, havingValue = &amp;quot;true&amp;quot;)
@ConditionalOnClass(name = &amp;quot;org.springframework.data.redis.core.RedisTemplate&amp;quot;)
@ConditionalOnMissingBean(CacheManager.class)
public CacheManager redisCacheManager() {
return new RedisCacheManager();
}</li>
</ul>
<p>/**</p>
<ul>
<li>Spring Boot 4 - 自定义条件
*/
@Bean
@ConditionalOnCustomCondition
public CustomService customService() {
return new CustomService();
}
</code></pre></li>
</ul>
<p>}</p>
<p>/**</p>
<ul>
<li>自定义条件注解
*/
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Conditional(CustomCondition.class)
@interface ConditionalOnCustomCondition {}</li>
</ul>
<p>/**</p>
<ul>
<li>
<p>自定义条件实现
*/
class CustomCondition implements org.springframework.context.annotation.Condition {
@Override
public boolean matches(
org.springframework.context.annotation.ConditionContext context,
org.springframework.core.type.AnnotatedTypeMetadata metadata) {</p>
<pre><code> Environment env = context.getEnvironment();
<p>// 复杂的条件逻辑
boolean isProduction = &amp;quot;production&amp;quot;.equals(env.getProperty(&amp;quot;app.environment&amp;quot;));
boolean hasRedis = context.getBeanFactory().getBeanNamesForType(
org.springframework.data.redis.core.RedisTemplate.class).length &amp;gt; 0;</p>
<p>return isProduction &amp;amp;&amp;amp; hasRedis;
</code></pre></p>
<p>}
}</p>
</li>
</ul>

<h2>3.4 启动性能优化</h2>
<h3>3.4.1 AOT (Ahead-of-Time) 编译</h3>
<p>Spring Boot 4 增强了 AOT 编译支持，显著提升启动性能。</p>
<h4>配置 AOT</h4>
<p><strong>pom.xml</strong>:</p>
<pre><code class="language-xml">&lt;build&gt;
    &lt;plugins&gt;
        &lt;plugin&gt;
            &lt;groupId&gt;org.springframework.boot&lt;/groupId&gt;
            &lt;artifactId&gt;spring-boot-maven-plugin&lt;/artifactId&gt;
            &lt;configuration&gt;
                &lt;aot&gt;
                    &lt;enabled&gt;true&lt;/enabled&gt;
                &lt;/aot&gt;
            &lt;/configuration&gt;
        &lt;/plugin&gt;
    &lt;/plugins&gt;
&lt;/build&gt;
</code></pre>
<p><strong>application.yml</strong>:</p>
<pre><code class="language-yaml">spring:
  aot:
    enabled: true
    processing:
      # 优化配置处理
      optimize-configuration: true
      # 移除未使用的自动配置
      remove-unused-autoconfig: true
</code></pre>
