          <div class="vditor-reset post__content"><h1>第26章：MySQL监控体系</h1>
<blockquote>
<p>监控是保障MySQL稳定运行的第一道防线</p>
</blockquote>
<h2>26.1 监控概述</h2>
<h3>26.1.1 为什么需要监控？</h3>
<p><strong>问题场景：</strong></p>
<ul>
<li>❌ 数据库突然变慢，不知道原因</li>
<li>❌ 磁盘空间满了，业务中断</li>
<li>❌ 主从延迟严重，数据不一致</li>
<li>❌ 连接数耗尽，无法连接</li>
</ul>
<p><strong>监控的目标：</strong></p>
<ul>
<li>✅ 及时发现问题</li>
<li>✅ 快速定位问题</li>
<li>✅ 预防故障发生</li>
<li>✅ 优化性能</li>
</ul>
<h3>26.1.2 监控指标分类</h3>
<p><strong>系统层监控：</strong></p>
<ul>
<li>CPU使用率</li>
<li>内存使用率</li>
<li>磁盘I/O</li>
<li>网络流量</li>
</ul>
<p><strong>MySQL层监控：</strong></p>
<ul>
<li>QPS/TPS</li>
<li>连接数</li>
<li>慢查询</li>
<li>主从延迟</li>
<li>Buffer Pool命中率</li>
</ul>
<p><strong>业务层监控：</strong></p>
<ul>
<li>订单量</li>
<li>用户活跃度</li>
<li>响应时间</li>
</ul>
<hr>
<h2>26.2 核心监控指标 ⭐⭐⭐⭐⭐</h2>
<h3>26.2.1 性能指标</h3>
<p><strong>1. QPS（每秒查询数）</strong></p>
<pre><code class="language-sql">-- 查看总查询数
SHOW GLOBAL STATUS LIKE 'Questions';
<p>-- 查看运行时间
SHOW GLOBAL STATUS LIKE 'Uptime';</p>
<p>-- 计算QPS
QPS = Questions / Uptime</p>
<p>-- 实时监控QPS
mysqladmin -u root -p -i 1 status | grep Queries
</code></pre></p>
<p><strong>2. TPS（每秒事务数）</strong></p>
<pre><code class="language-sql">-- 查看提交事务数
SHOW GLOBAL STATUS LIKE 'Com_commit';
<p>-- 查看回滚事务数
SHOW GLOBAL STATUS LIKE 'Com_rollback';</p>
<p>-- 计算TPS
TPS = (Com_commit + Com_rollback) / Uptime
</code></pre></p>
<p><strong>3. 响应时间</strong></p>
<pre><code class="language-sql">-- 查看慢查询
SHOW GLOBAL STATUS LIKE 'Slow_queries';
<p>-- 查看平均查询时间（需要开启慢查询日志）
-- 使用pt-query-digest分析
pt-query-digest /var/lib/mysql/slow.log
</code></pre></p>
<hr>
<h3>26.2.2 连接指标</h3>
<p><strong>1. 当前连接数</strong></p>
<pre><code class="language-sql">-- 查看当前连接数
SHOW STATUS LIKE 'Threads_connected';
<p>-- 查看正在运行的线程数
SHOW STATUS LIKE 'Threads_running';</p>
<p>-- 查看历史最大连接数
SHOW STATUS LIKE 'Max_used_connections';</p>
<p>-- 查看最大连接数配置
SHOW VARIABLES LIKE 'max_connections';</p>
<p>-- 连接使用率
连接使用率 = Threads_connected / max_connections * 100%
-- 建议：&lt;80%
</code></pre></p>
<p><strong>2. 连接错误</strong></p>
<pre><code class="language-sql">-- 查看连接错误数
SHOW STATUS LIKE 'Aborted_connects';
<p>-- 查看客户端异常断开数
SHOW STATUS LIKE 'Aborted_clients';</p>
<p>-- 查看连接错误次数
SHOW STATUS LIKE 'Connection_errors%';
</code></pre></p>
<p><strong>3. 线程缓存命中率</strong></p>
<pre><code class="language-sql">-- 查看创建的线程数
SHOW STATUS LIKE 'Threads_created';
<p>-- 查看总连接数
SHOW STATUS LIKE 'Connections';</p>
<p>-- 线程缓存命中率
命中率 = (1 - Threads_created / Connections) * 100%
-- 建议：&gt;90%
</code></pre></p>
<hr>
<h3>26.2.3 InnoDB指标</h3>
<p><strong>1. Buffer Pool命中率</strong></p>
<pre><code class="language-sql">-- 查看Buffer Pool读请求数
SHOW STATUS LIKE 'Innodb_buffer_pool_read_requests';
<p>-- 查看从磁盘读取的次数
SHOW STATUS LIKE 'Innodb_buffer_pool_reads';</p>
<p>-- Buffer Pool命中率
命中率 = (1 - Innodb_buffer_pool_reads / Innodb_buffer_pool_read_requests) * 100%
-- 建议：&gt;99%
</code></pre></p>
<p><strong>2. Buffer Pool使用率</strong></p>
<pre><code class="language-sql">-- 查看Buffer Pool总页数
SHOW STATUS LIKE 'Innodb_buffer_pool_pages_total';
<p>-- 查看空闲页数
SHOW STATUS LIKE 'Innodb_buffer_pool_pages_free';</p>
<p>-- 使用率
使用率 = (1 - pages_free / pages_total) * 100%
</code></pre></p>
<p><strong>3. InnoDB行操作</strong></p>
<pre><code class="language-sql">-- 查看行读取数
SHOW STATUS LIKE 'Innodb_rows_read';
<p>-- 查看行插入数
SHOW STATUS LIKE 'Innodb_rows_inserted';</p>
<p>-- 查看行更新数
SHOW STATUS LIKE 'Innodb_rows_updated';</p>
<p>-- 查看行删除数
SHOW STATUS LIKE 'Innodb_rows_deleted';
</code></pre></p>
<p><strong>4. InnoDB日志</strong></p>
<pre><code class="language-sql">-- 查看日志写入次数
SHOW STATUS LIKE 'Innodb_log_writes';
<p>-- 查看日志等待次数
SHOW STATUS LIKE 'Innodb_log_waits';
-- 如果&gt;0，说明日志缓冲区太小
</code></pre></p>
<hr>
<h3>26.2.4 复制指标</h3>
<p><strong>1. 主从延迟</strong></p>
<pre><code class="language-sql">-- 在从库上执行
SHOW SLAVE STATUS\G
<p>-- 关注字段：
-- Seconds_Behind_Master：延迟秒数
--   0：无延迟
--   NULL：复制未运行
--   &gt;0：有延迟</p>
<p>-- 建议：&lt;5秒
</code></pre></p>
<p><strong>2. 复制状态</strong></p>
<pre><code class="language-sql">SHOW SLAVE STATUS\G
<p>-- 关注字段：
-- Slave_IO_Running: Yes  -- IO线程运行
-- Slave_SQL_Running: Yes -- SQL线程运行
-- Last_Error: 空         -- 无错误
</code></pre></p>
<p><strong>3. 复制位置</strong></p>
<pre><code class="language-sql">-- 主库binlog位置
SHOW MASTER STATUS;
<p>-- 从库读取位置
SHOW SLAVE STATUS\G
-- Read_Master_Log_Pos
-- Exec_Master_Log_Pos
</code></pre></p>
<hr>
<h3>26.2.5 表和索引指标</h3>
<p><strong>1. 表锁</strong></p>
<pre><code class="language-sql">-- 查看表锁等待次数
SHOW STATUS LIKE 'Table_locks_waited';
<p>-- 查看表锁立即获取次数
SHOW STATUS LIKE 'Table_locks_immediate';</p>
<p>-- 表锁等待率
等待率 = Table_locks_waited / (Table_locks_waited + Table_locks_immediate) * 100%
-- 建议：&lt;1%
</code></pre></p>
<p><strong>2. 临时表</strong></p>
<pre><code class="language-sql">-- 查看创建的临时表数
SHOW STATUS LIKE 'Created_tmp_tables';
<p>-- 查看磁盘临时表数
SHOW STATUS LIKE 'Created_tmp_disk_tables';</p>
<p>-- 磁盘临时表比例
比例 = Created_tmp_disk_tables / Created_tmp_tables * 100%
-- 建议：&lt;25%
</code></pre></p>
<p><strong>3. 表缓存</strong></p>
<pre><code class="language-sql">-- 查看打开的表数
SHOW STATUS LIKE 'Open_tables';
<p>-- 查看打开表的次数
SHOW STATUS LIKE 'Opened_tables';</p>
<p>-- 查看表缓存大小
SHOW VARIABLES LIKE 'table_open_cache';</p>
<p>-- 如果Opened_tables持续增长，说明缓存不足
</code></pre></p>
<hr>
<h3>26.2.6 磁盘指标</h3>
<p><strong>1. 磁盘空间</strong></p>
<pre><code class="language-bash"># 查看磁盘使用情况
df -h
<h1>查看MySQL数据目录大小</h1>
<p>du -sh /var/lib/mysql</p>
<h1>查看各数据库大小</h1>
<p>SELECT
table_schema AS 'Database',
ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) AS 'Size (MB)'
FROM information_schema.TABLES
GROUP BY table_schema;</p>
<h1>查看各表大小</h1>
<p>SELECT
table_name AS 'Table',
ROUND(((data_length + index_length) / 1024 / 1024), 2) AS 'Size (MB)'
FROM information_schema.TABLES
WHERE table_schema = 'mydb'
ORDER BY (data_length + index_length) DESC;
</code></pre></p>
<p><strong>2. 磁盘I/O</strong></p>
<pre><code class="language-bash"># 使用iostat监控
iostat -x 1
<h1>关注指标：</h1>
<h1>%util：磁盘使用率（建议&lt;80%）</h1>
<h1>await：平均等待时间（SSD&lt;1ms，HDD&lt;10ms）</h1>
<h1>r/s, w/s：每秒读写次数</h1>
<p></code></pre></p>
<hr>

<p></code></pre></p>

<h3>26.3.2 图形化监控工具</h3>
<p><strong>1. Percona Monitoring and Management (PMM) ⭐⭐⭐⭐⭐</strong></p>
<p><strong>安装PMM Server：</strong></p>
<pre><code class="language-bash"># 使用Docker安装
docker run -d -p 80:80 \
  --name pmm-server \
  -v pmm-data:/srv \
  percona/pmm-server:2
<h1>访问Web界面</h1>
<p><a href="http://your-server-ip">http://your-server-ip</a></p>
<h1>默认用户名：admin</h1>
<h1>默认密码：admin</h1>
<p></code></pre></p>
<p><strong>安装PMM Client：</strong></p>
<pre><code class="language-bash"># CentOS
yum install pmm2-client
<h1>Ubuntu</h1>
<p>apt install pmm2-client</p>
<h1>配置PMM Server地址</h1>
<p>pmm-admin config --server-url=<a href="http://admin:admin@pmm-server:80">http://admin:admin@pmm-server:80</a></p>
<h1>添加MySQL监控</h1>
<p>pmm-admin add mysql --username=pmm --password=pmm_password --query-source=perfschema</p>
<h1>查看监控列表</h1>
<p>pmm-admin list
</code></pre></p>
<p><strong>PMM功能：</strong></p>
<ul>
<li>✅ MySQL性能监控</li>
<li>✅ 查询分析（Query Analytics）</li>
<li>✅ 慢查询分析</li>
<li>✅ 图形化展示</li>
<li>✅ 告警功能</li>
</ul>
<hr>
<p><strong>2. Prometheus + Grafana ⭐⭐⭐⭐⭐</strong></p>
<p><strong>安装mysqld_exporter：</strong></p>
<pre><code class="language-bash"># 下载
wget https://github.com/prometheus/mysqld_exporter/releases/download/v0.14.0/mysqld_exporter-0.14.0.linux-amd64.tar.gz
<h1>解压</h1>
<p>tar -xzf mysqld_exporter-0.14.0.linux-amd64.tar.gz
cd mysqld_exporter-0.14.0.linux-amd64</p>
<h1>创建MySQL监控用户</h1>
<p>mysql -u root -p
CREATE USER 'exporter'@'localhost' IDENTIFIED BY 'exporter_password';
GRANT PROCESS, REPLICATION CLIENT, SELECT ON <em>.</em> TO 'exporter'@'localhost';
FLUSH PRIVILEGES;</p>
<h1>创建配置文件</h1>
<p>cat &gt; .my.cnf &lt;&lt;EOF
[client]
user=exporter
password=exporter_password
EOF</p>
<h1>启动exporter</h1>
<p>./mysqld_exporter --config.my-cnf=.my.cnf &amp;</p>
<h1>访问metrics</h1>
<p>curl <a href="http://localhost:9104/metrics">http://localhost:9104/metrics</a>
</code></pre></p>
<p><strong>配置Prometheus：</strong></p>
<pre><code class="language-yaml"># prometheus.yml
scrape_configs:
  - job_name: 'mysql'
    static_configs:
      - targets: ['localhost:9104']
</code></pre>
<p><strong>Grafana导入MySQL仪表板：</strong></p>

<h2>26.5 监控脚本</h2>
<h3>26.5.1 连接数监控脚本</h3>
<pre><code class="language-bash">#!/bin/bash
# monitor_connections.sh
<h1>MySQL连接信息</h1>
<p>MYSQL_USER=&quot;root&quot;
MYSQL_PASS=&quot;password&quot;
MYSQL_HOST=&quot;localhost&quot;</p>
<h1>告警阈值</h1>
<p>WARNING_THRESHOLD=80  # 80%
CRITICAL_THRESHOLD=90 # 90%</p>
<h1>获取当前连接数</h1>
<p>CURRENT_CONN=$(mysql -u$MYSQL_USER -p$MYSQL_PASS -h$MYSQL_HOST -e &quot;SHOW STATUS LIKE 'Threads_connected';&quot; | awk 'NR==2 {print $2}')</p>
<h1>获取最大连接数</h1>
<p>MAX_CONN=$(mysql -u$MYSQL_USER -p$MYSQL_PASS -h$MYSQL_HOST -e &quot;SHOW VARIABLES LIKE 'max_connections';&quot; | awk 'NR==2 {print $2}')</p>
<h1>计算使用率</h1>
<p>USAGE=$(echo &quot;scale=2; $CURRENT_CONN / $MAX_CONN * 100&quot; | bc)</p>
<h1>判断告警</h1>
<p>if [ $(echo &quot;$USAGE &gt; $CRITICAL_THRESHOLD&quot; | bc) -eq 1 ]; then
echo &quot;CRITICAL: MySQL连接使用率${USAGE}%，当前连接数${CURRENT_CONN}，最大连接数${MAX_CONN}&quot;
# 发送告警
exit 2
elif [ $(echo &quot;$USAGE &gt; $WARNING_THRESHOLD&quot; | bc) -eq 1 ]; then
echo &quot;WARNING: MySQL连接使用率${USAGE}%，当前连接数${CURRENT_CONN}，最大连接数${MAX_CONN}&quot;
exit 1
else
echo &quot;OK: MySQL连接使用率${USAGE}%&quot;
exit 0
fi
</code></pre></p>
<h3>26.5.2 主从延迟监控脚本</h3>
<pre><code class="language-bash">#!/bin/bash
# monitor_replication.sh
<p>MYSQL_USER=&quot;root&quot;
MYSQL_PASS=&quot;password&quot;
MYSQL_HOST=&quot;localhost&quot;</p>
<h1>告警阈值（秒）</h1>
<p>WARNING_THRESHOLD=10
CRITICAL_THRESHOLD=60</p>
<h1>获取主从延迟</h1>
<p>DELAY=$(mysql -u$MYSQL_USER -p$MYSQL_PASS -h$MYSQL_HOST -e &quot;SHOW SLAVE STATUS\G&quot; | grep &quot;Seconds_Behind_Master&quot; | awk '{print $2}')</p>
<h1>检查复制状态</h1>
<p>IO_RUNNING=$(mysql -u$MYSQL_USER -p$MYSQL_PASS -h$MYSQL_HOST -e &quot;SHOW SLAVE STATUS\G&quot; | grep &quot;Slave_IO_Running&quot; | awk '{print $2}')
SQL_RUNNING=$(mysql -u$MYSQL_USER -p$MYSQL_PASS -h$MYSQL_HOST -e &quot;SHOW SLAVE STATUS\G&quot; | grep &quot;Slave_SQL_Running&quot; | awk '{print $2}')</p>
<h1>检查复制是否运行</h1>
<p>if [ &quot;$IO_RUNNING&quot; != &quot;Yes&quot; ] || [ &quot;$SQL_RUNNING&quot; != &quot;Yes&quot; ]; then
echo &quot;CRITICAL: MySQL复制未运行，IO_Running=$IO_RUNNING, SQL_Running=$SQL_RUNNING&quot;
exit 2
fi</p>
<h1>检查延迟</h1>
<p>if [ &quot;$DELAY&quot; == &quot;NULL&quot; ]; then
echo &quot;CRITICAL: MySQL复制延迟为NULL&quot;
exit 2
elif [ $DELAY -gt $CRITICAL_THRESHOLD ]; then
echo &quot;CRITICAL: MySQL主从延迟${DELAY}秒&quot;
exit 2
elif [ $DELAY -gt $WARNING_THRESHOLD ]; then
echo &quot;WARNING: MySQL主从延迟${DELAY}秒&quot;
exit 1
else
echo &quot;OK: MySQL主从延迟${DELAY}秒&quot;
exit 0
fi
</code></pre></p>
<h3>26.5.3 磁盘空间监控脚本</h3>

# monitor_disk.sh
<h1>数据目录</h1>

<h1>获取磁盘使用率</h1>
<p>USAGE=$(df -h $DATA_DIR | awk 'NR==2 {print $5}' | sed 's/%//')</p>
<h1>判断告警</h1>
<p>if [ $USAGE -gt $CRITICAL_THRESHOLD ]; then
echo &quot;CRITICAL: MySQL数据目录磁盘使用率${USAGE}%&quot;
exit 2
elif [ $USAGE -gt $WARNING_THRESHOLD ]; then
echo &quot;WARNING: MySQL数据目录磁盘使用率${USAGE}%&quot;
exit 1
else
echo &quot;OK: MySQL数据目录磁盘使用率${USAGE}%&quot;
exit 0
fi
</code></pre></p>
<h3>26.5.4 慢查询监控脚本</h3>
<pre><code class="language-bash">#!/bin/bash
# monitor_slow_queries.sh
<p>MYSQL_USER=&quot;root&quot;
MYSQL_PASS=&quot;password&quot;
MYSQL_HOST=&quot;localhost&quot;</p>

<h1>保存到临时文件</h1>
<p>TEMP_FILE=&quot;/tmp/slow_queries.tmp&quot;
if [ -f $TEMP_FILE ]; then
LAST_SLOW=$(cat $TEMP_FILE)
DIFF=$((CURRENT_SLOW - LAST_SLOW))</p>
<pre><code>if [ $DIFF -gt $THRESHOLD ]; then
    echo &amp;quot;WARNING: 最近1分钟产生${DIFF}个慢查询&amp;quot;

    echo &amp;quot;OK: 最近1分钟产生${DIFF}个慢查询&amp;quot;
    exit 0
fi
</code></pre>
<p>fi</p>
<h1>保存当前值</h1>
<p>echo $CURRENT_SLOW &gt; $TEMP_FILE
</code></pre></p>
