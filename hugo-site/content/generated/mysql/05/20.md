          <div class="vditor-reset post__content"><h1>第20章：MySQL服务器优化</h1>
<blockquote>
<p>合理的参数配置可以让MySQL性能提升数倍</p>
</blockquote>
<h2>20.1 MySQL配置文件</h2>
<h3>20.1.1 配置文件位置</h3>
<pre><code class="language-bash"># Linux配置文件位置（按优先级）
/etc/my.cnf
/etc/mysql/my.cnf
~/.my.cnf
--defaults-file指定的文件
<h1>查看配置文件位置</h1>
<p>mysql --help | grep my.cnf</p>
<h1>查看当前使用的配置</h1>
<p>mysql -u root -p -e &quot;SHOW VARIABLES&quot;</p>
<h1>查看配置文件</h1>
<p>cat /etc/my.cnf
</code></pre></p>
<h3>20.1.2 配置文件结构</h3>
<pre><code class="language-ini"># /etc/my.cnf
<p>[client]</p>
<h1>客户端配置</h1>
<p>port = 3306
socket = /var/lib/mysql/mysql.sock
default-character-set = utf8mb4</p>
<p>[mysql]</p>
<h1>mysql命令行工具配置</h1>
<p>no-auto-rehash
prompt = &quot;\u@\h [\d]&gt; &quot;</p>
<p>[mysqld]</p>
<h1>服务器配置（最重要）</h1>
<p>port = 3306
socket = /var/lib/mysql/mysql.sock
datadir = /var/lib/mysql
pid-file = /var/run/mysqld/mysqld.pid</p>
<h1>字符集</h1>
<p>character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci</p>
<h1>连接配置</h1>
<p>max_connections = 1000
max_connect_errors = 100000</p>
<h1>InnoDB配置</h1>
<p>innodb_buffer_pool_size = 8G
innodb_log_file_size = 512M</p>
<p>[mysqldump]</p>
<h1>mysqldump工具配置</h1>
<p>quick
max_allowed_packet = 64M
</code></pre></p>
<hr>
<h2>20.2 内存配置优化 ⭐⭐⭐⭐⭐</h2>
<h3>20.2.1 InnoDB Buffer Pool</h3>
<pre><code class="language-ini"># InnoDB Buffer Pool：最重要的内存配置
# 作用：缓存数据页和索引页
<p>[mysqld]</p>
<h1>大小设置（建议物理内存的50%-70%）</h1>
<p>innodb_buffer_pool_size = 8G</p>
<h1>实例数量（MySQL 5.5+）</h1>
<h1>建议：每个实例1GB，总大小/1GB</h1>
<p>innodb_buffer_pool_instances = 8</p>
<h1>预热Buffer Pool（MySQL 5.6+）</h1>
<h1>关闭时保存Buffer Pool状态，启动时加载</h1>
<p>innodb_buffer_pool_dump_at_shutdown = 1
innodb_buffer_pool_load_at_startup = 1</p>
<h1>查看Buffer Pool使用情况</h1>
<p>SHOW ENGINE INNODB STATUS\G</p>
<h1>查看Buffer Pool命中率</h1>
<p>SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool%';</p>
<h1>计算命中率</h1>
<h1>命中率 = (1 - Innodb_buffer_pool_reads / Innodb_buffer_pool_read_requests) * 100%</h1>
<h1>建议：命中率 &gt; 99%</h1>
<p></code></pre></p>
<h3>20.2.2 查询缓存（MySQL 5.7）</h3>
<pre><code class="language-ini"># 查询缓存（MySQL 8.0已移除）
[mysqld]
# 是否开启（建议关闭）
query_cache_type = 0
<h1>缓存大小</h1>
<p>query_cache_size = 0</p>
<h1>为什么建议关闭？</h1>
<h1>1. 任何表的更新都会清空该表的所有缓存</h1>
<h1>2. 并发性能差（全局锁）</h1>
<h1>3. 命中率低</h1>
<h1>4. MySQL 8.0已移除</h1>
<h1>查看查询缓存状态</h1>
<p>SHOW VARIABLES LIKE 'query_cache%';
SHOW STATUS LIKE 'Qcache%';
</code></pre></p>
<h3>20.2.3 线程缓存</h3>
<pre><code class="language-ini"># 线程缓存：缓存空闲的连接线程
[mysqld]
# 缓存的线程数
thread_cache_size = 64
<h1>查看线程缓存状态</h1>
<p>SHOW STATUS LIKE 'Threads%';</p>
<h1>Threads_created：创建的线程数</h1>
<h1>Threads_cached：缓存的线程数</h1>
<h1>Threads_connected：当前连接数</h1>
<h1>Threads_running：正在运行的线程数</h1>
<h1>如果Threads_created持续增长，说明thread_cache_size太小</h1>
<p></code></pre></p>
<h3>20.2.4 表缓存</h3>
<pre><code class="language-ini"># 表缓存：缓存打开的表
[mysqld]
# 缓存的表数量
table_open_cache = 4096
<h1>每个连接的表缓存</h1>
<p>table_open_cache_instances = 16</p>
<h1>查看表缓存状态</h1>
<p>SHOW STATUS LIKE 'Open%';</p>
<h1>Open_tables：当前打开的表数量</h1>
<h1>Opened_tables：历史打开的表数量</h1>
<h1>如果Opened_tables持续增长，说明table_open_cache太小</h1>
<p></code></pre></p>
<h3>20.2.5 排序缓冲区</h3>
<pre><code class="language-ini"># 排序缓冲区：ORDER BY、GROUP BY使用
[mysqld]
# 每个连接的排序缓冲区大小
sort_buffer_size = 2M
<h1>不要设置太大（会占用大量内存）</h1>
<h1>建议：256K - 4M</h1>
<h1>查看排序状态</h1>
<p>SHOW STATUS LIKE 'Sort%';</p>
<h1>Sort_merge_passes：多次排序的次数</h1>
<h1>如果该值很大，说明sort_buffer_size太小</h1>
<p></code></pre></p>
<h3>20.2.6 连接缓冲区</h3>
<pre><code class="language-ini"># 连接缓冲区：JOIN使用
[mysqld]
# 每个连接的JOIN缓冲区大小
join_buffer_size = 2M
<h1>建议：256K - 4M</h1>
<h1>读缓冲区</h1>
<p>read_buffer_size = 1M
read_rnd_buffer_size = 2M
</code></pre></p>
<hr>
<h2>20.3 连接配置优化</h2>
<h3>20.3.1 最大连接数</h3>
<pre><code class="language-ini">[mysqld]
# 最大连接数
max_connections = 1000
<h1>最大错误连接数</h1>
<p>max_connect_errors = 100000</p>
<h1>连接超时时间</h1>
<p>connect_timeout = 10</p>
<h1>交互式连接超时时间</h1>
<p>interactive_timeout = 28800  # 8小时</p>
<h1>非交互式连接超时时间</h1>
<p>wait_timeout = 28800  # 8小时</p>
<h1>查看当前连接数</h1>
<p>SHOW STATUS LIKE 'Threads_connected';
SHOW STATUS LIKE 'Max_used_connections';</p>
<h1>查看连接详情</h1>
<p>SHOW PROCESSLIST;
SELECT * FROM information_schema.PROCESSLIST;</p>
<h1>计算合理的max_connections</h1>
<h1>max_connections = (可用内存 - 系统内存 - Buffer Pool) / 单个连接内存</h1>
<h1>单个连接内存 ≈ sort_buffer_size + join_buffer_size + read_buffer_size + ...</h1>
<p></code></pre></p>
<h3>20.3.2 连接池配置</h3>
<pre><code class="language-ini"># 线程池（MySQL Enterprise或Percona Server）
[mysqld]
thread_handling = pool-of-threads
thread_pool_size = 16  # CPU核心数
thread_pool_max_threads = 1000
<h1>优势：</h1>
<h1>✅ 减少线程创建开销</h1>
<h1>✅ 提高并发性能</h1>
<h1>✅ 减少上下文切换</h1>
<p></code></pre></p>
<hr>
<h2>20.4 InnoDB配置优化 ⭐⭐⭐⭐⭐</h2>
<h3>20.4.1 Redo Log配置</h3>
<pre><code class="language-ini">[mysqld]
# Redo Log文件大小
innodb_log_file_size = 512M
<h1>Redo Log文件数量</h1>
<p>innodb_log_files_in_group = 2</p>
<h1>Redo Log缓冲区大小</h1>
<p>innodb_log_buffer_size = 16M</p>
<h1>Redo Log刷新策略（最重要）</h1>
<p>innodb_flush_log_at_trx_commit = 1</p>
<h1>innodb_flush_log_at_trx_commit的值：</h1>
<h1>0：每秒刷新一次（性能最好，可能丢失1秒数据）</h1>
<h1>1：每次事务提交刷新（最安全，性能最差）⭐ 推荐</h1>
<h1>2：每次事务提交写入OS缓存，每秒刷新（折中）</h1>
<h1>性能对比：</h1>
<h1>0: 1000 TPS</h1>
<h1>1: 100 TPS</h1>
<h1>2: 500 TPS</h1>
<h1>建议：</h1>
<h1>生产环境：1（数据安全第一）</h1>
<h1>开发环境：2（性能和安全平衡）</h1>
<h1>测试环境：0（性能优先）</h1>
<p></code></pre></p>
<h3>20.4.2 Binlog配置</h3>
<pre><code class="language-ini">[mysqld]
# 开启binlog
log_bin = mysql-bin
<h1>binlog格式</h1>
<p>binlog_format = ROW  # ROW/STATEMENT/MIXED</p>
<h1>binlog缓冲区大小</h1>
<p>binlog_cache_size = 4M</p>
<h1>binlog刷新策略</h1>
<p>sync_binlog = 1</p>
<h1>sync_binlog的值：</h1>
<h1>0：不主动刷新，由OS决定（性能最好，可能丢失数据）</h1>
<h1>1：每次事务提交刷新（最安全，性能最差）⭐ 推荐</h1>
<h1>N：每N次事务提交刷新（折中）</h1>
<h1>binlog过期时间</h1>
<p>expire_logs_days = 7  # MySQL 5.7
binlog_expire_logs_seconds = 604800  # MySQL 8.0（7天）</p>
<h1>建议：</h1>
<h1>生产环境：sync_binlog = 1</h1>
<h1>开发环境：sync_binlog = 0</h1>
<p></code></pre></p>
<h3>20.4.3 双1配置</h3>
<pre><code class="language-ini"># 双1配置：保证数据安全
[mysqld]
innodb_flush_log_at_trx_commit = 1
sync_binlog = 1
<h1>优点：</h1>
<h1>✅ 数据绝对安全</h1>
<h1>✅ 不会丢失任何已提交的事务</h1>
<h1>缺点：</h1>
<h1>❌ 性能下降（每次提交都要刷盘）</h1>
<h1>性能优化：</h1>
<h1>1. 使用SSD（减少I/O延迟）</h1>
<h1>2. 使用RAID卡缓存（带电池）</h1>
<h1>3. 批量提交事务</h1>
<p></code></pre></p>
<h3>20.4.4 刷新策略</h3>
<pre><code class="language-ini">[mysqld]
# 脏页刷新策略
innodb_flush_method = O_DIRECT
<h1>innodb_flush_method的值：</h1>
<h1>fsync：默认值</h1>
<h1>O_DIRECT：绕过OS缓存，直接写磁盘（推荐）⭐</h1>
<h1>O_DSYNC：同步写数据文件</h1>
<h1>O_DIRECT的优势：</h1>
<h1>✅ 避免双重缓冲（OS缓存 + Buffer Pool）</h1>
<h1>✅ 减少内存占用</h1>
<h1>✅ 提高性能</h1>
<h1>脏页刷新比例</h1>
<p>innodb_max_dirty_pages_pct = 75  # 脏页达到75%时开始刷新</p>
<h1>自适应刷新</h1>
<p>innodb_adaptive_flushing = 1</p>
<h1>查看脏页比例</h1>
<p>SHOW STATUS LIKE 'Innodb_buffer_pool_pages_dirty';
SHOW STATUS LIKE 'Innodb_buffer_pool_pages_total';
</code></pre></p>
<h3>20.4.5 I/O配置</h3>
<pre><code class="language-ini">[mysqld]
# I/O线程数
innodb_read_io_threads = 4
innodb_write_io_threads = 4
<h1>建议：</h1>
<h1>SSD：8-16</h1>
<h1>HDD：4-8</h1>
<h1>I/O容量</h1>
<p>innodb_io_capacity = 200  # HDD
innodb_io_capacity = 2000  # SSD</p>
<h1>最大I/O容量</h1>
<p>innodb_io_capacity_max = 2000  # HDD
innodb_io_capacity_max = 4000  # SSD</p>
<h1>查看I/O状态</h1>
<p>SHOW ENGINE INNODB STATUS\G
</code></pre></p>
<h3>20.4.6 其他InnoDB配置</h3>
<pre><code class="language-ini">[mysqld]
# 每表一个文件
innodb_file_per_table = 1  # 推荐开启
<h1>自动扩展</h1>
<p>innodb_autoextend_increment = 64  # 每次扩展64MB</p>
<h1>锁等待超时</h1>
<p>innodb_lock_wait_timeout = 50  # 50秒</p>
<h1>死锁检测</h1>
<p>innodb_deadlock_detect = 1</p>
<h1>在线DDL</h1>
<p>innodb_online_alter_log_max_size = 128M</p>
<h1>统计信息</h1>
<p>innodb_stats_persistent = 1  # 持久化统计信息
innodb_stats_auto_recalc = 1  # 自动更新统计信息
</code></pre></p>
<hr>
<h2>20.5 慢查询日志配置</h2>
<h3>20.5.1 开启慢查询日志</h3>
<pre><code class="language-ini">[mysqld]
# 开启慢查询日志
slow_query_log = 1
<h1>慢查询日志文件</h1>
<p>slow_query_log_file = /var/lib/mysql/slow.log</p>
<h1>慢查询阈值（秒）</h1>
<p>long_query_time = 1</p>
<h1>记录未使用索引的查询</h1>
<p>log_queries_not_using_indexes = 1</p>
<h1>限制每分钟记录的未使用索引的查询数量</h1>
<p>log_throttle_queries_not_using_indexes = 10</p>
<h1>记录慢的管理语句（ALTER TABLE等）</h1>
<p>log_slow_admin_statements = 1</p>
<h1>最小扫描行数</h1>
<p>min_examined_row_limit = 1000</p>
<h1>查看慢查询状态</h1>
<p>SHOW VARIABLES LIKE 'slow_query%';
SHOW STATUS LIKE 'Slow_queries';
</code></pre></p>
<h3>20.5.2 分析慢查询日志</h3>
<pre><code class="language-bash"># 使用mysqldumpslow分析
mysqldumpslow -s t -t 10 /var/lib/mysql/slow.log
# -s t：按查询时间排序
# -t 10：显示前10条
<h1>使用pt-query-digest分析（推荐）</h1>
<p>pt-query-digest /var/lib/mysql/slow.log</p>
<h1>输出到文件</h1>
<p>pt-query-digest /var/lib/mysql/slow.log &gt; slow_report.txt</p>
<h1>只分析指定时间范围</h1>
<p>pt-query-digest --since '2024-11-10 00:00:00' <br>
--until '2024-11-10 23:59:59' <br>
/var/lib/mysql/slow.log
</code></pre></p>

<h2>20.6 其他重要配置</h2>

<h1>常用SQL模式：</h1>

<h1>ERROR_FOR_DIVISION_BY_ZERO：除以0报错</h1>

<h1>查看SQL模式</h1>

<h1>临时修改</h1>
<p>SET SESSION sql_mode = 'STRICT_TRANS_TABLES';
SET GLOBAL sql_mode = 'STRICT_TRANS_TABLES';
</code></pre></p>

# 服务器字符集

</code></pre></p>

<h2>20.7 性能监控</h2>
<h3>20.7.1 查看系统状态</h3>
<pre><code class="language-sql">-- 查看所有状态变量
SHOW GLOBAL STATUS;
<p>-- 查看连接状态
SHOW STATUS LIKE 'Threads%';
SHOW STATUS LIKE 'Connections';
SHOW STATUS LIKE 'Max_used_connections';</p>
<p>-- 查看InnoDB状态
SHOW ENGINE INNODB STATUS\G</p>
<p>-- 查看慢查询
SHOW STATUS LIKE 'Slow_queries';</p>
<p>-- 查看表锁
SHOW STATUS LIKE 'Table_locks%';</p>
<p>-- 查看临时表
SHOW STATUS LIKE 'Created_tmp%';
</code></pre></p>
<h3>20.7.2 性能指标</h3>
<pre><code class="language-sql">-- 1. QPS（每秒查询数）
SHOW GLOBAL STATUS LIKE 'Questions';
SHOW GLOBAL STATUS LIKE 'Uptime';
-- QPS = Questions / Uptime
<p>-- 2. TPS（每秒事务数）
SHOW GLOBAL STATUS LIKE 'Com_commit';
SHOW GLOBAL STATUS LIKE 'Com_rollback';
-- TPS = (Com_commit + Com_rollback) / Uptime</p>
<p>-- 3. Buffer Pool命中率
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_read_requests';
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_reads';
-- 命中率 = (1 - reads / read_requests) * 100%</p>
<p>-- 4. 线程缓存命中率
SHOW GLOBAL STATUS LIKE 'Threads_created';
SHOW GLOBAL STATUS LIKE 'Connections';
-- 命中率 = (1 - Threads_created / Connections) * 100%</p>
<p>-- 5. 表缓存命中率
SHOW GLOBAL STATUS LIKE 'Opened_tables';
SHOW GLOBAL STATUS LIKE 'Open_tables';
-- 如果Opened_tables持续增长，说明缓存不足
</code></pre></p>
<h3>20.7.3 性能诊断</h3>
<pre><code class="language-sql">-- 查看当前正在执行的查询
SHOW PROCESSLIST;
<p>-- 查看锁等待
SELECT * FROM information_schema.INNODB_LOCKS;
SELECT * FROM information_schema.INNODB_LOCK_WAITS;</p>
<p>-- 查看事务
SELECT * FROM information_schema.INNODB_TRX;</p>
<p>-- 查看表统计信息
SELECT * FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'mydb';</p>
<p>-- 查看索引统计信息
SELECT * FROM information_schema.STATISTICS
WHERE TABLE_SCHEMA = 'mydb';
</code></pre></p>
