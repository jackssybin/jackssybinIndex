          <div class="vditor-reset post__content"><h1>第21章：硬件与操作系统优化</h1>
<blockquote>
<p>硬件是性能的基础，操作系统是性能的保障</p>
</blockquote>
<h2>21.1 硬件选型</h2>
<h3>21.1.1 CPU选择</h3>
<p><strong>CPU核心数：</strong></p>
<ul>
<li>✅ 多核心优于高频率</li>
<li>✅ 推荐：8核心以上</li>
<li>✅ MySQL可以利用多核心并行处理</li>
</ul>
<p><strong>CPU架构：</strong></p>
<ul>
<li>✅ x86_64架构</li>
<li>✅ Intel Xeon或AMD EPYC</li>
<li>✅ 支持AES-NI（加密加速）</li>
</ul>
<p><strong>性能指标：</strong></p>
<pre><code class="language-bash"># 查看CPU信息
lscpu
cat /proc/cpuinfo
<h1>查看CPU核心数</h1>
<p>grep -c processor /proc/cpuinfo</p>
<h1>查看CPU型号</h1>
<p>grep &quot;model name&quot; /proc/cpuinfo | head -1</p>
<h1>CPU性能测试</h1>
<p>sysbench cpu --cpu-max-prime=20000 run
</code></pre></p>
<p><strong>建议：</strong></p>
<ul>
<li>小型：4核心</li>
<li>中型：8-16核心</li>
<li>大型：16-32核心</li>
</ul>
<hr>
<h3>21.1.2 内存选择</h3>
<p><strong>内存大小：</strong></p>
<ul>
<li>✅ 越大越好</li>
<li>✅ 推荐：物理内存 &gt; 数据库大小 * 1.5</li>
<li>✅ Buffer Pool = 物理内存 * 60%-70%</li>
</ul>
<p><strong>内存类型：</strong></p>
<ul>
<li>✅ DDR4或DDR5</li>
<li>✅ ECC内存（纠错内存）</li>
<li>✅ 高频率（2666MHz+）</li>
</ul>
<p><strong>性能指标：</strong></p>
<pre><code class="language-bash"># 查看内存信息
free -h
cat /proc/meminfo
<h1>查看内存详细信息</h1>
<p>dmidecode -t memory</p>
<h1>内存性能测试</h1>
<p>sysbench memory --memory-total-size=10G run
</code></pre></p>
<p><strong>建议：</strong></p>
<ul>
<li>小型：8-16GB</li>
<li>中型：32-64GB</li>
<li>大型：128GB+</li>
</ul>
<hr>
<h3>21.1.3 磁盘选择 ⭐⭐⭐⭐⭐</h3>
<p><strong>磁盘类型对比：</strong></p>
<table>
<thead>
<tr>
<th>类型</th>
<th>IOPS</th>
<th>延迟</th>
<th>价格</th>
<th>推荐度</th>
</tr>
</thead>
<tbody>
<tr>
<td>HDD</td>
<td>100-200</td>
<td>10-20ms</td>
<td>低</td>
<td>❌ 不推荐</td>
</tr>
<tr>
<td>SATA SSD</td>
<td>10K-50K</td>
<td>0.1-1ms</td>
<td>中</td>
<td>⭐⭐⭐</td>
</tr>
<tr>
<td>NVMe SSD</td>
<td>100K-1M</td>
<td>0.01-0.1ms</td>
<td>高</td>
<td>⭐⭐⭐⭐⭐</td>
</tr>
</tbody>
</table>
<p><strong>SSD的优势：</strong></p>
<ul>
<li>✅ IOPS高（100倍于HDD）</li>
<li>✅ 延迟低（1/100于HDD）</li>
<li>✅ 随机读写性能好</li>
<li>✅ 适合数据库</li>
</ul>
<p><strong>RAID配置：</strong></p>
<ul>
<li><strong>RAID 0</strong>：性能最好，无冗余（不推荐）</li>
<li><strong>RAID 1</strong>：镜像，冗余好，性能一般</li>
<li><strong>RAID 5</strong>：性能和冗余平衡（写性能差）</li>
<li><strong>RAID 10</strong>：性能和冗余都好（推荐）⭐⭐⭐⭐⭐</li>
</ul>
<p><strong>磁盘性能测试：</strong></p>
<pre><code class="language-bash"># 顺序读写测试
dd if=/dev/zero of=/tmp/test bs=1M count=1024
dd if=/tmp/test of=/dev/null bs=1M
<h1>随机读写测试（推荐）</h1>
<p>sysbench fileio --file-total-size=10G prepare
sysbench fileio --file-total-size=10G <br>
--file-test-mode=rndrw <br>
--time=60 <br>
--max-requests=0 run
sysbench fileio --file-total-size=10G cleanup</p>
<h1>使用fio测试</h1>
<p>fio -filename=/tmp/test -direct=1 -iodepth 1 -thread <br>
-rw=randrw -ioengine=psync -bs=16k -size=10G <br>
-numjobs=10 -runtime=60 -group_reporting -name=mytest
</code></pre></p>
<p><strong>建议：</strong></p>
<ul>
<li>数据文件：NVMe SSD + RAID 10</li>
<li>日志文件：NVMe SSD</li>
<li>备份文件：HDD（成本低）</li>
</ul>
<hr>
<h3>21.1.4 网络选择</h3>
<p><strong>网卡：</strong></p>
<ul>
<li>✅ 千兆网卡（1Gbps）：小型</li>
<li>✅ 万兆网卡（10Gbps）：中大型</li>
<li>✅ 双网卡绑定（提高带宽和可用性）</li>
</ul>
<p><strong>网络性能测试：</strong></p>
<pre><code class="language-bash"># 测试网络带宽
iperf3 -s  # 服务端
iperf3 -c server_ip  # 客户端
<h1>测试网络延迟</h1>
<p>ping server_ip</p>
<h1>测试MySQL网络性能</h1>
<p>sysbench oltp_read_write <br>
--mysql-host=remote_host <br>
--mysql-user=root <br>
--mysql-password=password <br>
--mysql-db=test <br>
--tables=10 <br>
--table-size=100000 <br>
prepare
</code></pre></p>
<hr>
<h2>21.2 操作系统选择</h2>
<h3>21.2.1 Linux发行版选择</h3>
<p><strong>推荐发行版：</strong></p>
<ul>
<li>✅ <strong>CentOS/RHEL</strong>：企业级，稳定（推荐）⭐⭐⭐⭐⭐</li>
<li>✅ <strong>Ubuntu Server</strong>：易用，社区活跃</li>
<li>✅ <strong>Debian</strong>：稳定</li>
<li>❌ Windows：不推荐（性能差）</li>
</ul>
<p><strong>版本选择：</strong></p>
<ul>
<li>CentOS 7/8 或 RHEL 7/8</li>
<li>Ubuntu 20.04 LTS 或 22.04 LTS</li>
</ul>
<p><strong>查看系统信息：</strong></p>
<pre><code class="language-bash"># 查看发行版
cat /etc/os-release
lsb_release -a
<h1>查看内核版本</h1>
<p>uname -r</p>
<h1>查看系统位数</h1>
<p>uname -m
</code></pre></p>
<hr>
<h2>21.3 操作系统优化 ⭐⭐⭐⭐⭐</h2>
<h3>21.3.1 文件系统选择</h3>
<p><strong>文件系统对比：</strong></p>
<table>
<thead>
<tr>
<th>文件系统</th>
<th>性能</th>
<th>稳定性</th>
<th>推荐度</th>
</tr>
</thead>
<tbody>
<tr>
<td>ext3</td>
<td>一般</td>
<td>好</td>
<td>❌</td>
</tr>
<tr>
<td>ext4</td>
<td>好</td>
<td>好</td>
<td>⭐⭐⭐⭐</td>
</tr>
<tr>
<td>XFS</td>
<td>很好</td>
<td>很好</td>
<td>⭐⭐⭐⭐⭐</td>
</tr>
<tr>
<td>Btrfs</td>
<td>好</td>
<td>一般</td>
<td>⭐⭐⭐</td>
</tr>
</tbody>
</table>
<p><strong>推荐：XFS</strong></p>
<ul>
<li>✅ 高性能</li>
<li>✅ 支持大文件</li>
<li>✅ 日志文件系统</li>
<li>✅ CentOS 7默认文件系统</li>
</ul>
<p><strong>挂载选项优化：</strong></p>
<pre><code class="language-bash"># /etc/fstab
/dev/sdb1 /var/lib/mysql xfs noatime,nodiratime,nobarrier 0 0
<h1>选项说明：</h1>
<h1>noatime：不更新访问时间（提高性能）</h1>
<h1>nodiratime：不更新目录访问时间</h1>
<h1>nobarrier：禁用写屏障（SSD或有RAID卡缓存时）</h1>
<h1>重新挂载</h1>
<p>mount -o remount /var/lib/mysql</p>
<h1>查看挂载选项</h1>
<p>mount | grep mysql
</code></pre></p>
<hr>
<h3>21.3.2 I/O调度器优化</h3>
<p><strong>I/O调度器类型：</strong></p>
<ul>
<li><strong>CFQ</strong>（Completely Fair Queuing）：默认，适合HDD</li>
<li><strong>Deadline</strong>：适合数据库</li>
<li><strong>NOOP</strong>：适合SSD/NVMe（推荐）⭐⭐⭐⭐⭐</li>
</ul>
<p><strong>查看和修改I/O调度器：</strong></p>
<pre><code class="language-bash"># 查看当前I/O调度器
cat /sys/block/sda/queue/scheduler
<h1>临时修改</h1>
<p>echo noop &gt; /sys/block/sda/queue/scheduler</p>
<h1>永久修改（/etc/default/grub）</h1>
<p>GRUB_CMDLINE_LINUX=&quot;elevator=noop&quot;</p>
<h1>更新grub</h1>
<p>grub2-mkconfig -o /boot/grub2/grub.cfg</p>
<h1>重启生效</h1>
<p>reboot
</code></pre></p>
<p><strong>建议：</strong></p>
<ul>
<li>SSD/NVMe：noop</li>
<li>HDD：deadline</li>
</ul>
<hr>
<h3>21.3.3 内核参数优化</h3>
<p><strong>优化内核参数（/etc/sysctl.conf）：</strong></p>
<pre><code class="language-bash"># 网络优化
net.core.somaxconn = 65535  # 最大连接队列
net.core.netdev_max_backlog = 65535  # 网卡接收队列
net.ipv4.tcp_max_syn_backlog = 65535  # SYN队列
net.ipv4.tcp_fin_timeout = 10  # FIN超时时间
net.ipv4.tcp_tw_reuse = 1  # 重用TIME_WAIT连接
net.ipv4.tcp_tw_recycle = 0  # 不回收TIME_WAIT（避免NAT问题）
net.ipv4.ip_local_port_range = 1024 65535  # 端口范围
<h1>内存优化</h1>
<p>vm.swappiness = 0  # 禁用swap（重要）⭐⭐⭐⭐⭐
vm.dirty_ratio = 10  # 脏页比例
vm.dirty_background_ratio = 5  # 后台刷新脏页比例</p>
<h1>文件系统优化</h1>
<p>fs.file-max = 6553560  # 最大文件句柄数
fs.aio-max-nr = 1048576  # 异步I/O最大数</p>
<h1>应用配置</h1>
<p>sysctl -p  # 立即生效</p>
<h1>查看配置</h1>
<p>sysctl -a | grep swappiness
</code></pre></p>
<p><strong>重点：禁用swap</strong></p>
<pre><code class="language-bash"># 为什么要禁用swap？
# MySQL使用swap会导致性能急剧下降
<h1>临时禁用</h1>
<p>swapoff -a</p>
<h1>永久禁用（注释/etc/fstab中的swap行）</h1>
<h1>/dev/mapper/centos-swap swap swap defaults 0 0</h1>
<h1>查看swap使用情况</h1>
<p>free -h
swapon -s
</code></pre></p>
<hr>
<h3>21.3.4 文件句柄限制</h3>
<p><strong>修改文件句柄限制（/etc/security/limits.conf）：</strong></p>
<pre><code class="language-bash"># 添加以下内容
mysql soft nofile 65535
mysql hard nofile 65535
mysql soft nproc 65535
mysql hard nproc 65535
<h1>或者</h1>
<ul>
<li>soft nofile 65535</li>
<li>hard nofile 65535</li>
<li>soft nproc 65535</li>
<li>hard nproc 65535</li>
</ul>
<h1>查看当前限制</h1>
<p>ulimit -n
ulimit -u</p>
<h1>查看MySQL进程的限制</h1>
<p>cat /proc/$(pidof mysqld)/limits
</code></pre></p>
<p><strong>修改systemd服务限制（/etc/systemd/system/mysqld.service.d/limits.conf）：</strong></p>
<pre><code class="language-ini">[Service]
LimitNOFILE=65535
LimitNPROC=65535
<h1>重新加载</h1>
<p>systemctl daemon-reload
systemctl restart mysqld
</code></pre></p>
<hr>
<h3>21.3.5 透明大页（THP）</h3>
<p><strong>禁用透明大页：</strong></p>
<pre><code class="language-bash"># 为什么要禁用？
# THP会导致MySQL性能下降和内存碎片
<h1>查看THP状态</h1>
<p>cat /sys/kernel/mm/transparent_hugepage/enabled</p>
<h1>临时禁用</h1>
<p>echo never &gt; /sys/kernel/mm/transparent_hugepage/enabled
echo never &gt; /sys/kernel/mm/transparent_hugepage/defrag</p>
<h1>永久禁用（/etc/rc.local）</h1>
<p>echo never &gt; /sys/kernel/mm/transparent_hugepage/enabled
echo never &gt; /sys/kernel/mm/transparent_hugepage/defrag
chmod +x /etc/rc.d/rc.local</p>
<h1>或者修改grub（/etc/default/grub）</h1>
<p>GRUB_CMDLINE_LINUX=&quot;transparent_hugepage=never&quot;
grub2-mkconfig -o /boot/grub2/grub.cfg
reboot
</code></pre></p>
<hr>
<h3>21.3.6 NUMA优化</h3>
<p><strong>什么是NUMA？</strong></p>
<ul>
<li>Non-Uniform Memory Access</li>
<li>多CPU系统中，每个CPU有自己的本地内存</li>
<li>访问本地内存快，访问远程内存慢</li>
</ul>
<p><strong>查看NUMA信息：</strong></p>
<pre><code class="language-bash"># 查看NUMA节点
numactl --hardware
<h1>查看MySQL进程的NUMA分布</h1>
<p>numastat -p $(pidof mysqld)
</code></pre></p>
<p><strong>NUMA优化：</strong></p>
<pre><code class="language-bash"># 方法1：禁用NUMA（推荐）
# /etc/default/grub
GRUB_CMDLINE_LINUX=&quot;numa=off&quot;
grub2-mkconfig -o /boot/grub2/grub.cfg
reboot
<h1>方法2：使用numactl启动MySQL</h1>
<p>numactl --interleave=all mysqld_safe &amp;</p>
<h1>方法3：在my.cnf中配置</h1>
<p>[mysqld_safe]
numa-interleave = 1
</code></pre></p>

<ol>

<li>使用RAID 10</li>
</ol>

<ul>
<li>%util降到30%</li>
<li>await降到1ms</li>
<li>查询速度提升10倍</li>
</ul>

</ol>
<p><strong>效果：</strong></p>
<ul>
<li>swap使用为0</li>

</ul>
