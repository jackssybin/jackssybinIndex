<p>构建安全的数据库环境</p>
</blockquote>
<h2>30.1 安全加固概述</h2>
<h3>30.1.1 安全威胁</h3>
<p><strong>常见威胁：</strong></p>
<ul>
<li>SQL注入攻击</li>
<li>暴力破解</li>
<li>权限滥用</li>
<li>数据泄露</li>
<li>拒绝服务攻击（DoS）</li>
</ul>
<hr>
<h2>30.2 账户安全 ⭐⭐⭐⭐⭐</h2>
<h3>30.2.1 删除默认账户</h3>
<p><strong>删除匿名用户和测试数据库：</strong></p>
<pre><code class="language-sql">-- 查看所有用户
SELECT user, host FROM mysql.user;
<p>-- 删除匿名用户
DELETE FROM mysql.user WHERE user = '';</p>
<p>-- 删除test数据库
DROP DATABASE IF EXISTS test;</p>
<p>-- 刷新权限
FLUSH PRIVILEGES;
</code></pre></p>
<h3>30.2.2 禁用root远程登录</h3>
<p><strong>只允许root本地登录：</strong></p>
<pre><code class="language-sql">-- 查看root用户
SELECT user, host FROM mysql.user WHERE user = 'root';
<p>-- 删除root远程登录
DELETE FROM mysql.user WHERE user = 'root' AND host != 'localhost';</p>
<p>-- 或修改root只能本地登录
UPDATE mysql.user SET host = 'localhost' WHERE user = 'root';</p>
<p>FLUSH PRIVILEGES;
</code></pre></p>
<h3>30.2.3 强密码策略</h3>
<p><strong>配置密码策略：</strong></p>
<pre><code class="language-ini"># my.cnf
[mysqld]
# 密码验证插件
validate_password.policy = STRONG
validate_password.length = 12
validate_password.mixed_case_count = 1
validate_password.number_count = 1
validate_password.special_char_count = 1
<h1>密码过期</h1>
<p>default_password_lifetime = 90</p>
<h1>密码重用限制</h1>
<p>password_history = 5
password_reuse_interval = 365
</code></pre></p>
<p><strong>修改用户密码：</strong></p>
<pre><code class="language-sql">-- 设置密码过期
ALTER USER 'app_user'@'localhost' PASSWORD EXPIRE INTERVAL 90 DAY;
<p>-- 设置密码历史
ALTER USER 'app_user'@'localhost' PASSWORD HISTORY 5;</p>
<p>-- 设置密码重用间隔
ALTER USER 'app_user'@'localhost' PASSWORD REUSE INTERVAL 365 DAY;
</code></pre></p>
<h3>30.2.4 限制登录失败次数</h3>
<p><strong>配置登录失败锁定：</strong></p>
<pre><code class="language-sql">-- MySQL 8.0.19+
ALTER USER 'app_user'@'localhost' 
FAILED_LOGIN_ATTEMPTS 3 
PASSWORD_LOCK_TIME 1;  -- 锁定1天
<p>-- 解锁用户
ALTER USER 'app_user'@'localhost' ACCOUNT UNLOCK;
</code></pre></p>
<hr>
<h2>30.3 网络安全 ⭐⭐⭐⭐⭐</h2>
<h3>30.3.1 绑定IP地址</h3>
<p><strong>只监听内网IP：</strong></p>
<pre><code class="language-ini"># my.cnf
[mysqld]
bind-address = 192.168.1.10
<h1>或只监听本地</h1>
<p>bind-address = 127.0.0.1
</code></pre></p>
<h3>30.3.2 修改默认端口</h3>
<p><strong>修改默认端口（3306）：</strong></p>
<pre><code class="language-ini"># my.cnf
[mysqld]
port = 13306
</code></pre>
<h3>30.3.3 防火墙配置</h3>
<p><strong>配置防火墙：</strong></p>
<pre><code class="language-bash"># CentOS/RHEL
# 只允许指定IP访问MySQL
firewall-cmd --permanent --add-rich-rule='rule family=&quot;ipv4&quot; source address=&quot;192.168.1.0/24&quot; port protocol=&quot;tcp&quot; port=&quot;3306&quot; accept'
firewall-cmd --reload
<h1>Ubuntu</h1>
<h1>只允许指定IP访问MySQL</h1>
<p>ufw allow from 192.168.1.0/24 to any port 3306
ufw enable
</code></pre></p>
<hr>
<h2>30.4 SSL/TLS加密 ⭐⭐⭐⭐</h2>
<h3>30.4.1 生成SSL证书</h3>
<p><strong>生成证书：</strong></p>
<pre><code class="language-bash"># 使用mysql_ssl_rsa_setup生成证书（MySQL 5.7+）
mysql_ssl_rsa_setup --datadir=/var/lib/mysql
<h1>查看生成的证书</h1>
<p>ls -l /var/lib/mysql/*.pem</p>
<h1>ca.pem：CA证书</h1>
<h1>server-cert.pem：服务器证书</h1>
<h1>server-key.pem：服务器私钥</h1>
<h1>client-cert.pem：客户端证书</h1>
<h1>client-key.pem：客户端私钥</h1>
<p></code></pre></p>
<h3>30.4.2 配置SSL</h3>
<p><strong>服务器配置：</strong></p>
<pre><code class="language-ini"># my.cnf
[mysqld]
ssl-ca = /var/lib/mysql/ca.pem
ssl-cert = /var/lib/mysql/server-cert.pem
ssl-key = /var/lib/mysql/server-key.pem
<h1>要求所有连接使用SSL</h1>
<p>require_secure_transport = ON
</code></pre></p>
<p><strong>查看SSL状态：</strong></p>
<pre><code class="language-sql">-- 查看SSL变量
SHOW VARIABLES LIKE '%ssl%';
<p>-- 查看当前连接是否使用SSL
SHOW STATUS LIKE 'Ssl_cipher';
</code></pre></p>
<h3>30.4.3 要求用户使用SSL</h3>
<p><strong>创建要求SSL的用户：</strong></p>
<pre><code class="language-sql">-- 创建用户并要求SSL
CREATE USER 'secure_user'@'%' IDENTIFIED BY 'Password@123' REQUIRE SSL;
<p>-- 修改现有用户要求SSL
ALTER USER 'app_user'@'%' REQUIRE SSL;</p>
<p>-- 要求X509证书
ALTER USER 'app_user'@'%' REQUIRE X509;
</code></pre></p>
<p><strong>客户端连接：</strong></p>
<pre><code class="language-bash"># 使用SSL连接
mysql -u secure_user -p \
  --ssl-ca=/path/to/ca.pem \
  --ssl-cert=/path/to/client-cert.pem \
  --ssl-key=/path/to/client-key.pem
</code></pre>
<hr>
<h2>30.5 审计日志 ⭐⭐⭐⭐</h2>
<h3>30.5.1 开启审计日志</h3>
<p><strong>使用audit_log插件（MySQL Enterprise）：</strong></p>
<pre><code class="language-sql">-- 安装插件
INSTALL PLUGIN audit_log SONAME 'audit_log.so';
<p>-- 配置
SET GLOBAL audit_log_policy = ALL;
SET GLOBAL audit_log_format = JSON;
</code></pre></p>
<p><strong>使用general_log（不推荐生产环境）：</strong></p>
<pre><code class="language-sql">-- 开启general_log
SET GLOBAL general_log = ON;
SET GLOBAL general_log_file = '/var/log/mysql/general.log';
<p>-- 查看日志
tail -f /var/log/mysql/general.log
</code></pre></p>
<h3>30.5.2 监控可疑活动</h3>
<p><strong>监控登录失败：</strong></p>
<pre><code class="language-bash"># 查看错误日志中的登录失败
grep &quot;Access denied&quot; /var/log/mysql/error.log
</code></pre>
<hr>
<h2>30.6 数据加密 ⭐⭐⭐⭐</h2>
<h3>30.6.1 表空间加密</h3>
<p><strong>开启表空间加密（MySQL 5.7+）：</strong></p>
<pre><code class="language-sql">-- 创建加密表
CREATE TABLE sensitive_data (
    id INT PRIMARY KEY,
    credit_card VARCHAR(100)
) ENCRYPTION = 'Y';
<p>-- 修改现有表为加密表
ALTER TABLE users ENCRYPTION = 'Y';
</code></pre></p>
<p><strong>配置加密密钥：</strong></p>
<pre><code class="language-ini"># my.cnf
[mysqld]
early-plugin-load = keyring_file.so
keyring_file_data = /var/lib/mysql-keyring/keyring
</code></pre>
<h3>30.6.2 binlog加密</h3>
<p><strong>开启binlog加密（MySQL 8.0.14+）：</strong></p>
<pre><code class="language-sql">SET GLOBAL binlog_encryption = ON;
</code></pre>
<hr>
<h2>30.7 SQL注入防护 ⭐⭐⭐⭐⭐</h2>
<h3>30.7.1 使用预处理语句</h3>
<p><strong>预处理语句（推荐）：</strong></p>
<pre><code class="language-sql">-- ✅ 使用预处理语句（安全）
PREPARE stmt FROM 'SELECT * FROM users WHERE id = ?';
SET @id = 1;
EXECUTE stmt USING @id;
<p>-- 应用程序中使用参数化查询
-- Java: PreparedStatement
-- Python: cursor.execute(&quot;SELECT * FROM users WHERE id = %s&quot;, (user_id,))
-- PHP: $stmt-&gt;bind_param(&quot;i&quot;, $user_id);
</code></pre></p>
<p><strong>避免拼接SQL：</strong></p>
<pre><code class="language-sql">-- ❌ 不安全：SQL拼接
SELECT * FROM users WHERE name = 'admin' OR '1'='1';
<p>-- ✅ 安全：预处理语句
PREPARE stmt FROM 'SELECT * FROM users WHERE name = ?';
</code></pre></p>
<hr>
<h2>30.8 安全配置清单</h2>
<h3>30.8.1 my.cnf安全配置</h3>
<p><strong>推荐配置：</strong></p>
<pre><code class="language-ini">[mysqld]
# 1. 网络安全
bind-address = 192.168.1.10
port = 13306
<h1>2. SSL/TLS</h1>
<p>ssl-ca = /var/lib/mysql/ca.pem
ssl-cert = /var/lib/mysql/server-cert.pem
ssl-key = /var/lib/mysql/server-key.pem
require_secure_transport = ON</p>
<h1>3. 密码策略</h1>
<p>validate_password.policy = STRONG
validate_password.length = 12
default_password_lifetime = 90</p>
<h1>4. 禁用危险功能</h1>
<p>local_infile = OFF
skip-symbolic-links</p>
<h1>5. 日志</h1>
<p>log_error = /var/log/mysql/error.log
slow_query_log = ON
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2</p>
<h1>6. 限制</h1>
<p>max_connections = 500
max_connect_errors = 10
</code></pre></p>
