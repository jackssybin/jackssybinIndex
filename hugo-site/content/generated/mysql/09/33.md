<blockquote>
<p>常见问题的终极解决方案</p>

<h2>33.1 性能问题</h2>
<h3>33.1.1 查询突然变慢</h3>
<p><strong>问题现象：</strong></p>
<pre><code>原本很快的查询突然变慢
</code></pre>
<p><strong>排查步骤：</strong></p>
<pre><code class="language-sql">-- 1. 查看是否有锁等待
SHOW PROCESSLIST;
<p>-- 2. 查看是否有慢查询
SELECT * FROM information_schema.PROCESSLIST WHERE TIME &gt; 10;</p>
<p>-- 3. 查看表统计信息是否过期
SHOW TABLE STATUS LIKE 'users';</p>
<p>-- 4. 分析执行计划
EXPLAIN SELECT * FROM users WHERE age &gt; 18;
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 更新统计信息
ANALYZE TABLE users;
<p>-- 2. 重建索引
ALTER TABLE users DROP INDEX idx_age, ADD INDEX idx_age(age);</p>
<p>-- 3. 优化表
OPTIMIZE TABLE users;</p>
<p>-- 4. 检查是否需要添加索引
CREATE INDEX idx_age ON users(age);
</code></pre></p>
<hr>
<h3>33.1.2 Buffer Pool命中率低</h3>
<p><strong>问题现象：</strong></p>
<pre><code>Buffer Pool命中率&lt;90%，性能差
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-sql">-- 查看Buffer Pool命中率
SHOW STATUS LIKE 'Innodb_buffer_pool_read%';
<p>-- 计算命中率
-- 命中率 = (Innodb_buffer_pool_read_requests - Innodb_buffer_pool_reads) / Innodb_buffer_pool_read_requests * 100%
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 增加Buffer Pool大小（50-70%内存）
SET GLOBAL innodb_buffer_pool_size = 8G;
<p>-- 或修改配置文件
[mysqld]
innodb_buffer_pool_size = 8G
innodb_buffer_pool_instances = 8
</code></pre></p>
<hr>
<h3>33.1.3 临时表过多</h3>
<p><strong>问题现象：</strong></p>
<pre><code>Created_tmp_disk_tables很高
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-sql">-- 查看临时表统计
SHOW STATUS LIKE 'Created_tmp%';
<p>-- Created_tmp_tables：内存临时表
-- Created_tmp_disk_tables：磁盘临时表（慢）
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 增加临时表大小
SET GLOBAL tmp_table_size = 64M;
SET GLOBAL max_heap_table_size = 64M;
<p>-- 2. 优化SQL，避免临时表
-- ❌ 使用DISTINCT
SELECT DISTINCT name FROM users;</p>
<p>-- ✅ 使用GROUP BY
SELECT name FROM users GROUP BY name;</p>
<p>-- 3. 添加索引
CREATE INDEX idx_name ON users(name);
</code></pre></p>
<hr>
<h2>33.2 数据问题</h2>
<h3>33.2.1 主键冲突</h3>
<p><strong>问题现象：</strong></p>
<pre><code>ERROR 1062 (23000): Duplicate entry '1' for key 'PRIMARY'
</code></pre>
<p><strong>原因：</strong></p>
<pre><code>1. 手动插入了主键值
2. 自增值被重置
3. 主从复制导致
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 查看当前自增值
SHOW CREATE TABLE users;
<p>-- 2. 修改自增值
ALTER TABLE users AUTO_INCREMENT = 1000;</p>
<p>-- 3. 查找最大ID
SELECT MAX(id) FROM users;</p>
<p>-- 4. 设置自增值为最大ID+1
ALTER TABLE users AUTO_INCREMENT = (SELECT MAX(id) + 1 FROM users);
</code></pre></p>
<hr>
<h3>33.2.2 数据不一致</h3>
<p><strong>问题现象：</strong></p>
<pre><code>主从数据不一致
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-bash"># 使用pt-table-checksum检查
pt-table-checksum \
  --host=master_host \
  --user=root \
  --password=password \
  --databases=mydb
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-bash"># 使用pt-table-sync同步
pt-table-sync \
  --execute \
  --sync-to-master \
  h=slave_host,D=mydb,t=users
</code></pre>
<hr>
<h3>33.2.3 误删除数据</h3>
<p><strong>问题现象：</strong></p>
<pre><code>DELETE语句误删除了数据
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-bash"># 1. 立即停止应用，防止继续写入
<h1>2. 使用binlog恢复</h1>
<h1>找到误删除之前的binlog位置</h1>
<p>mysqlbinlog --start-datetime=&quot;2024-11-10 09:00:00&quot; <br>
--stop-datetime=&quot;2024-11-10 09:59:59&quot; <br>
/var/lib/mysql/mysql-bin.000010 &gt; recover.sql</p>
<h1>3. 过滤掉DELETE语句</h1>
<p>grep -v &quot;DELETE FROM users&quot; recover.sql &gt; recover_filtered.sql</p>
<h1>4. 恢复数据</h1>
<p>mysql -u root -p mydb &lt; recover_filtered.sql</p>
<h1>5. 验证数据</h1>
<p>SELECT COUNT(*) FROM users;
</code></pre></p>
<hr>
<h2>33.3 连接问题</h2>
<h3>33.3.1 连接数耗尽</h3>
<p><strong>问题现象：</strong></p>
<pre><code>ERROR 1040 (HY000): Too many connections
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-sql">-- 查看当前连接数
SHOW STATUS LIKE 'Threads_connected';
<p>-- 查看最大连接数
SHOW VARIABLES LIKE 'max_connections';</p>
<p>-- 查看连接来源
SELECT USER, HOST, COUNT(*)
FROM information_schema.PROCESSLIST
GROUP BY USER, HOST;
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 临时增加最大连接数
SET GLOBAL max_connections = 1000;
<p>-- 2. 杀死空闲连接
SELECT CONCAT('KILL ', id, ';')
FROM information_schema.PROCESSLIST
WHERE COMMAND = 'Sleep' AND TIME &gt; 600;</p>
<p>-- 3. 设置超时时间
SET GLOBAL wait_timeout = 600;
SET GLOBAL interactive_timeout = 600;</p>
<p>-- 4. 应用层使用连接池
</code></pre></p>
<hr>
<h3>33.3.2 连接超时</h3>
<p><strong>问题现象：</strong></p>
<pre><code>ERROR 2013 (HY000): Lost connection to MySQL server during query
</code></pre>
<p><strong>原因：</strong></p>
<pre><code>1. 网络问题
2. 查询时间过长
3. max_allowed_packet太小
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 增加超时时间
SET GLOBAL wait_timeout = 28800;
SET GLOBAL interactive_timeout = 28800;
<p>-- 2. 增加max_allowed_packet
SET GLOBAL max_allowed_packet = 64M;</p>
<p>-- 3. 优化慢查询
</code></pre></p>
<hr>
<h2>33.4 复制问题</h2>
<h3>33.4.1 主从延迟</h3>
<p><strong>问题现象：</strong></p>
<pre><code>Seconds_Behind_Master &gt; 10
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-sql">-- 查看主从状态
SHOW SLAVE STATUS\G
<p>-- 关注字段：
-- Seconds_Behind_Master：延迟秒数
-- Slave_SQL_Running_State：SQL线程状态
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 开启并行复制（MySQL 5.7+）
STOP SLAVE;
SET GLOBAL slave_parallel_type = 'LOGICAL_CLOCK';
SET GLOBAL slave_parallel_workers = 4;
START SLAVE;
<p>-- 2. 升级从库硬件</p>
<p>-- 3. 拆分大事务</p>
<p>-- 4. 优化慢查询
</code></pre></p>
<hr>
<h3>33.4.2 复制中断</h3>
<p><strong>问题现象：</strong></p>
<pre><code>Slave_IO_Running: No 或 Slave_SQL_Running: No
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-sql">SHOW SLAVE STATUS\G
<p>-- 查看错误信息：
-- Last_IO_Error
-- Last_SQL_Error
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 跳过错误（谨慎使用）
STOP SLAVE;
SET GLOBAL sql_slave_skip_counter = 1;
START SLAVE;
<p>-- 2. 重新搭建主从</p>
<h1>主库备份</h1>
<p>mysqldump -u root -p --single-transaction --master-data=2 --all-databases &gt; backup.sql</p>
<h1>从库导入</h1>
<p>mysql -u root -p &lt; backup.sql</p>
<h1>配置复制</h1>
<p>CHANGE MASTER TO ...;
START SLAVE;
</code></pre></p>
<hr>
<h2>33.5 磁盘问题</h2>
<h3>33.5.1 磁盘空间满</h3>
<p><strong>问题现象：</strong></p>
<pre><code>ERROR 3 (HY000): Error writing file (Errcode: 28 - No space left on device)
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-bash"># 查看磁盘空间
df -h
<h1>查看MySQL数据目录大小</h1>
<p>du -sh /var/lib/mysql/*
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 清理binlog
PURGE BINARY LOGS BEFORE DATE_SUB(NOW(), INTERVAL 7 DAY);
<p>-- 2. 清理慢查询日志
&gt; /var/lib/mysql/slow.log</p>
<p>-- 3. 删除旧分区
ALTER TABLE orders DROP PARTITION p2021;</p>
<p>-- 4. 优化表
OPTIMIZE TABLE users;
</code></pre></p>
<hr>
<h3>33.5.2 I/O性能差</h3>
<p><strong>问题现象：</strong></p>
<pre><code>iostat显示I/O使用率100%
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-bash"># 查看I/O
iostat -x 1
<h1>查看哪个进程占用I/O</h1>
<p>iotop
</code></pre></p>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 优化慢查询
<p>-- 2. 增加Buffer Pool
SET GLOBAL innodb_buffer_pool_size = 8G;</p>
<p>-- 3. 使用SSD</p>
<p>-- 4. 调整刷盘策略
SET GLOBAL innodb_flush_log_at_trx_commit = 2;
SET GLOBAL sync_binlog = 0;
</code></pre></p>
<hr>
<h2>33.6 字符集问题</h2>
<h3>33.6.1 中文乱码</h3>
<p><strong>问题现象：</strong></p>
<pre><code>中文显示为???或乱码
</code></pre>
<p><strong>排查：</strong></p>
<pre><code class="language-sql">-- 查看字符集
SHOW VARIABLES LIKE 'character%';
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 设置字符集
SET NAMES utf8mb4;
<p>-- 2. 修改表字符集
ALTER TABLE users CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;</p>
<p>-- 3. 修改配置文件
[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci</p>
<p>[client]
default-character-set = utf8mb4
</code></pre></p>
<hr>
<h3>33.6.2 emoji无法存储</h3>
<p><strong>问题现象：</strong></p>
<pre><code>ERROR 1366 (HY000): Incorrect string value: '\xF0\x9F...'
</code></pre>
<p><strong>原因：</strong></p>
<pre><code>使用utf8而不是utf8mb4
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 转换为utf8mb4
ALTER TABLE users CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
</code></pre>
<hr>
<h2>33.7 安全问题</h2>
<h3>33.7.1 SQL注入</h3>
<p><strong>问题现象：</strong></p>
<pre><code>恶意SQL：SELECT * FROM users WHERE id = 1 OR 1=1
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- ✅ 使用预处理语句
PREPARE stmt FROM 'SELECT * FROM users WHERE id = ?';
SET @id = 1;
EXECUTE stmt USING @id;
<p>-- 应用程序中使用参数化查询
-- Java: PreparedStatement
-- Python: cursor.execute(&quot;SELECT * FROM users WHERE id = %s&quot;, (user_id,))
</code></pre></p>
<hr>
<h3>33.7.2 暴力破解</h3>
<p><strong>问题现象：</strong></p>
<pre><code>大量登录失败
</code></pre>
<p><strong>解决方案：</strong></p>
<pre><code class="language-sql">-- 1. 限制登录失败次数（MySQL 8.0.19+）
ALTER USER 'app_user'@'localhost' 
FAILED_LOGIN_ATTEMPTS 3 
PASSWORD_LOCK_TIME 1;
<p>-- 2. 配置防火墙</p>
<p>-- 3. 使用fail2ban
</code></pre></p>

</ol>
