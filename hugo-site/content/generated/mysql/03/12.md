          <div class="vditor-reset post__content"><h1>第12章：触发器与事件</h1>
<blockquote>
<p>自动化数据库操作的利器</p>
</blockquote>
<h2>12.1 触发器（Trigger）⭐⭐⭐⭐</h2>
<h3>12.1.1 什么是触发器？</h3>
<p><strong>定义：</strong></p>
<ul>
<li>在特定事件发生时自动执行的SQL语句</li>
<li>事件：INSERT、UPDATE、DELETE</li>
<li>时机：BEFORE、AFTER</li>
</ul>
<p><strong>语法：</strong></p>
<pre><code class="language-sql">CREATE TRIGGER trigger_name
{BEFORE | AFTER} {INSERT | UPDATE | DELETE}
ON table_name
FOR EACH ROW
BEGIN
    -- SQL语句
END;
</code></pre>
<h3>12.1.2 触发器示例</h3>
<p><strong>BEFORE INSERT触发器：</strong></p>
<pre><code class="language-sql">-- 自动设置创建时间
DELIMITER $$
<p>CREATE TRIGGER tr_users_before_insert
BEFORE INSERT ON users
FOR EACH ROW
BEGIN
SET NEW.create_time = NOW();
END$$</p>
<p>DELIMITER ;
</code></pre></p>
<p><strong>AFTER INSERT触发器：</strong></p>
<pre><code class="language-sql">-- 插入用户后，记录日志
DELIMITER $$
<p>CREATE TRIGGER tr_users_after_insert
AFTER INSERT ON users
FOR EACH ROW
BEGIN
INSERT INTO user_logs(user_id, action, create_time)
VALUES (<a href="http://NEW.id">NEW.id</a>, 'INSERT', NOW());
END$$</p>
<p>DELIMITER ;
</code></pre></p>
<p><strong>BEFORE UPDATE触发器：</strong></p>
<pre><code class="language-sql">-- 更新前记录旧值
DELIMITER $$
<p>CREATE TRIGGER tr_users_before_update
BEFORE UPDATE ON users
FOR EACH ROW
BEGIN
SET NEW.update_time = NOW();
INSERT INTO user_history(user_id, old_name, new_name, update_time)
VALUES (<a href="http://OLD.id">OLD.id</a>, <a href="http://OLD.name">OLD.name</a>, <a href="http://NEW.name">NEW.name</a>, NOW());
END$$</p>
<p>DELIMITER ;
</code></pre></p>
<p><strong>AFTER DELETE触发器：</strong></p>
<pre><code class="language-sql">-- 删除用户后，删除相关订单
DELIMITER $$
<p>CREATE TRIGGER tr_users_after_delete
AFTER DELETE ON users
FOR EACH ROW
BEGIN
DELETE FROM orders WHERE user_id = <a href="http://OLD.id">OLD.id</a>;
END$$</p>
<p>DELIMITER ;
</code></pre></p>
<h3>12.1.3 NEW和OLD</h3>
<p><strong>NEW：</strong></p>
<ul>
<li>INSERT：新插入的行</li>
<li>UPDATE：更新后的行</li>
<li>DELETE：不可用</li>
</ul>
<p><strong>OLD：</strong></p>
<ul>
<li>INSERT：不可用</li>
<li>UPDATE：更新前的行</li>
<li>DELETE：被删除的行</li>
</ul>
<h3>12.1.4 触发器管理</h3>
<p><strong>查看触发器：</strong></p>
<pre><code class="language-sql">-- 查看所有触发器
SHOW TRIGGERS;
<p>-- 查看指定表的触发器
SHOW TRIGGERS WHERE <code>Table</code> = 'users';</p>
<p>-- 查看触发器定义
SHOW CREATE TRIGGER tr_users_before_insert;
</code></pre></p>
<p><strong>删除触发器：</strong></p>
<pre><code class="language-sql">DROP TRIGGER IF EXISTS tr_users_before_insert;
</code></pre>
<h3>12.1.5 触发器注意事项</h3>
<p><strong>限制：</strong></p>
<ul>
<li>❌ 不能在触发器中调用存储过程返回结果集</li>
<li>❌ 不能使用CALL语句</li>
<li>❌ 不能使用事务控制语句（COMMIT、ROLLBACK）</li>
</ul>
<p><strong>性能影响：</strong></p>
<ul>
<li>⚠️ 触发器会影响INSERT、UPDATE、DELETE性能</li>
<li>⚠️ 避免在触发器中执行复杂操作</li>
</ul>
<hr>
<h2>12.2 事件（Event）⭐⭐⭐⭐</h2>
<h3>12.2.1 什么是事件？</h3>
<p><strong>定义：</strong></p>
<ul>
<li>定时任务，类似Linux的cron</li>
<li>在指定时间自动执行SQL语句</li>
</ul>
<p><strong>开启事件调度器：</strong></p>
<pre><code class="language-sql">-- 查看事件调度器状态
SHOW VARIABLES LIKE 'event_scheduler';
<p>-- 开启事件调度器
SET GLOBAL event_scheduler = ON;</p>
<p>-- 配置文件中开启
[mysqld]
event_scheduler = ON
</code></pre></p>
<h3>12.2.2 创建事件</h3>
<p><strong>一次性事件：</strong></p>
<pre><code class="language-sql">-- 在指定时间执行一次
CREATE EVENT evt_delete_old_logs
ON SCHEDULE AT '2024-12-31 23:59:59'
DO
    DELETE FROM logs WHERE create_time &lt; DATE_SUB(NOW(), INTERVAL 1 YEAR);
</code></pre>
<p><strong>周期性事件：</strong></p>
<pre><code class="language-sql">-- 每天凌晨1点执行
CREATE EVENT evt_daily_cleanup
ON SCHEDULE EVERY 1 DAY
STARTS '2024-01-01 01:00:00'
DO
    DELETE FROM logs WHERE create_time &lt; DATE_SUB(NOW(), INTERVAL 30 DAY);
<p>-- 每小时执行
CREATE EVENT evt_hourly_task
ON SCHEDULE EVERY 1 HOUR
DO
UPDATE statistics SET update_time = NOW();</p>
<p>-- 每5分钟执行
CREATE EVENT evt_5min_task
ON SCHEDULE EVERY 5 MINUTE
DO
CALL sp_update_cache();
</code></pre></p>
<p><strong>带结束时间的事件：</strong></p>
<pre><code class="language-sql">CREATE EVENT evt_temp_task
ON SCHEDULE EVERY 1 DAY
STARTS '2024-01-01 00:00:00'
ENDS '2024-12-31 23:59:59'
DO
    DELETE FROM temp_data WHERE create_time &lt; DATE_SUB(NOW(), INTERVAL 1 DAY);
</code></pre>
<h3>12.2.3 事件管理</h3>
<p><strong>查看事件：</strong></p>
<pre><code class="language-sql">-- 查看所有事件
SHOW EVENTS;
<p>-- 查看指定数据库的事件
SHOW EVENTS FROM mydb;</p>
<p>-- 查看事件定义
SHOW CREATE EVENT evt_daily_cleanup;
</code></pre></p>
<p><strong>修改事件：</strong></p>
<pre><code class="language-sql">-- 禁用事件
ALTER EVENT evt_daily_cleanup DISABLE;
<p>-- 启用事件
ALTER EVENT evt_daily_cleanup ENABLE;</p>
<p>-- 修改事件
ALTER EVENT evt_daily_cleanup
ON SCHEDULE EVERY 2 DAY;
</code></pre></p>
<p><strong>删除事件：</strong></p>
<pre><code class="language-sql">DROP EVENT IF EXISTS evt_daily_cleanup;
</code></pre>
<h3>12.2.4 事件实战案例</h3>
<p><strong>案例1：定期清理日志</strong></p>
<pre><code class="language-sql">CREATE EVENT evt_cleanup_logs
ON SCHEDULE EVERY 1 DAY
STARTS '2024-01-01 02:00:00'
DO
BEGIN
    -- 删除30天前的日志
    DELETE FROM access_logs WHERE create_time &lt; DATE_SUB(NOW(), INTERVAL 30 DAY);
    DELETE FROM error_logs WHERE create_time &lt; DATE_SUB(NOW(), INTERVAL 30 DAY);
<pre><code>-- 优化表
OPTIMIZE TABLE access_logs;
OPTIMIZE TABLE error_logs;
</code></pre>
<p>END;
</code></pre></p>
<p><strong>案例2：定期备份数据</strong></p>
<pre><code class="language-sql">CREATE EVENT evt_backup_data
ON SCHEDULE EVERY 1 DAY
STARTS '2024-01-01 03:00:00'
DO
BEGIN
    -- 备份到历史表
    INSERT INTO orders_history
    SELECT * FROM orders 
    WHERE create_time &lt; DATE_SUB(NOW(), INTERVAL 1 YEAR);
<pre><code>-- 删除旧数据
DELETE FROM orders 
WHERE create_time &amp;lt; DATE_SUB(NOW(), INTERVAL 1 YEAR);
</code></pre>
<p>END;
</code></pre></p>
<p><strong>案例3：定期更新统计数据</strong></p>
<pre><code class="language-sql">CREATE EVENT evt_update_statistics
ON SCHEDULE EVERY 1 HOUR
DO
BEGIN
    -- 更新用户统计
    UPDATE user_statistics us
    SET 
        order_count = (SELECT COUNT(*) FROM orders WHERE user_id = us.user_id),
        total_amount = (SELECT IFNULL(SUM(amount), 0) FROM orders WHERE user_id = us.user_id),
        update_time = NOW();
END;
</code></pre>

<h2>12.3 本章总结</h2>
<p><strong>本章学习内容：</strong></p>
<ul>
<li>✅ <strong>触发器</strong>（BEFORE/AFTER、INSERT/UPDATE/DELETE）⭐⭐⭐⭐</li>
<li>✅ <strong>事件</strong>（一次性、周期性）⭐⭐⭐⭐</li>
<li>✅ NEW和OLD</li>
<li>✅ 实战案例</li>
</ul>
<p><strong>重点掌握：</strong></p>
<ol>
<li><strong>触发器：BEFORE/AFTER + INSERT/UPDATE/DELETE</strong></li>
<li><strong>NEW：新数据，OLD：旧数据</strong></li>
<li><strong>事件：定时任务</strong></li>
<li><strong>event_scheduler = ON</strong></li>
</ol>
<p><strong>触发器时机：</strong></p>
<ul>
<li>BEFORE：在操作之前执行</li>
<li>AFTER：在操作之后执行</li>
</ul>
<p><strong>事件调度：</strong></p>
<ul>
<li>AT：一次性</li>
<li>EVERY：周期性</li>
</ul>
<p><strong>面试重点：</strong></p>
<ul>
<li>触发器的作用</li>
<li>NEW和OLD的区别</li>
<li>事件和cron的区别</li>
<li>触发器的性能影响</li>
</ul>
