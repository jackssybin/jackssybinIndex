          <div class="vditor-reset post__content"><h1>第14章：字符集与排序规则</h1>
<blockquote>
<p>正确处理中文和emoji的关键</p>
</blockquote>
<h2>14.1 字符集概述</h2>
<h3>14.1.1 什么是字符集？</h3>
<p><strong>定义：</strong></p>
<ul>
<li>字符集（Character Set）：字符的编码方式</li>
<li>排序规则（Collation）：字符的比较规则</li>
</ul>
<p><strong>常用字符集：</strong></p>
<ul>
<li><code>latin1</code>：西欧字符（MySQL默认，已过时）</li>
<li><code>gbk</code>：简体中文</li>
<li><code>utf8</code>：UTF-8编码（最多3字节，不支持emoji）⚠️</li>
<li><code>utf8mb4</code>：UTF-8编码（最多4字节，支持emoji）✅ <strong>推荐</strong></li>
</ul>
<hr>
<h2>14.2 字符集详解 ⭐⭐⭐⭐⭐</h2>
<h3>14.2.1 utf8 vs utf8mb4</h3>
<p><strong>重要区别：</strong></p>
<pre><code class="language-sql">-- utf8：最多3字节，不支持emoji
-- utf8mb4：最多4字节，支持emoji
<p>-- ❌ utf8无法存储emoji
CREATE TABLE test_utf8 (
content VARCHAR(100) CHARACTER SET utf8
);
INSERT INTO test_utf8 VALUES ('😀');  -- 报错</p>
<p>-- ✅ utf8mb4可以存储emoji
CREATE TABLE test_utf8mb4 (
content VARCHAR(100) CHARACTER SET utf8mb4
);
INSERT INTO test_utf8mb4 VALUES ('😀');  -- 成功
</code></pre></p>
<p><strong>推荐：</strong></p>
<ul>
<li>✅ <strong>统一使用utf8mb4</strong></li>
<li>❌ 不要使用utf8</li>
</ul>
<h3>14.2.2 查看字符集</h3>
<p><strong>查看系统字符集：</strong></p>
<pre><code class="language-sql">-- 查看所有字符集
SHOW CHARACTER SET;
<p>-- 查看字符集变量
SHOW VARIABLES LIKE 'character%';</p>
<p>-- 输出：
-- character_set_client: utf8mb4      -- 客户端字符集
-- character_set_connection: utf8mb4  -- 连接字符集
-- character_set_database: utf8mb4    -- 数据库字符集
-- character_set_results: utf8mb4     -- 结果字符集
-- character_set_server: utf8mb4      -- 服务器字符集
-- character_set_system: utf8         -- 系统字符集（固定）
</code></pre></p>
<p><strong>查看表字符集：</strong></p>
<pre><code class="language-sql">-- 查看表字符集
SHOW CREATE TABLE users;
<p>-- 查看列字符集
SHOW FULL COLUMNS FROM users;
</code></pre></p>
<h3>14.2.3 设置字符集</h3>
<p><strong>服务器级别：</strong></p>
<pre><code class="language-ini"># my.cnf
[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
<p>[client]
default-character-set = utf8mb4
</code></pre></p>
<p><strong>数据库级别：</strong></p>
<pre><code class="language-sql">-- 创建数据库时指定
CREATE DATABASE mydb 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
<p>-- 修改数据库字符集
ALTER DATABASE mydb
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
</code></pre></p>
<p><strong>表级别：</strong></p>
<pre><code class="language-sql">-- 创建表时指定
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100)
) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
<p>-- 修改表字符集
ALTER TABLE users
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
</code></pre></p>
<p><strong>列级别：</strong></p>
<pre><code class="language-sql">-- 创建列时指定
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci
);
<p>-- 修改列字符集
ALTER TABLE users
MODIFY COLUMN name VARCHAR(100)
CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
</code></pre></p>
<hr>
<h2>14.3 排序规则（Collation）⭐⭐⭐⭐</h2>
<h3>14.3.1 常用排序规则</h3>
<p><strong>utf8mb4排序规则：</strong></p>
<ul>
<li><code>utf8mb4_general_ci</code>：通用，不区分大小写，性能好</li>
<li><code>utf8mb4_unicode_ci</code>：Unicode标准，区分重音，性能稍差 ✅ <strong>推荐</strong></li>
<li><code>utf8mb4_bin</code>：二进制，区分大小写</li>
</ul>
<p><strong>示例：</strong></p>
<pre><code class="language-sql">-- utf8mb4_general_ci：不区分大小写
SELECT 'A' = 'a' COLLATE utf8mb4_general_ci;  -- 1（相等）
<p>-- utf8mb4_bin：区分大小写
SELECT 'A' = 'a' COLLATE utf8mb4_bin;  -- 0（不相等）
</code></pre></p>
<h3>14.3.2 排序规则对比</h3>
<p><strong>示例：</strong></p>
<pre><code class="language-sql">-- 创建测试表
CREATE TABLE test_collation (
    name VARCHAR(100)
) CHARACTER SET utf8mb4;
<p>INSERT INTO test_collation VALUES ('A'), ('a'), ('B'), ('b');</p>
<p>-- utf8mb4_general_ci：不区分大小写
SELECT * FROM test_collation ORDER BY name COLLATE utf8mb4_general_ci;
-- 结果：A, a, B, b 或 a, A, b, B</p>
<p>-- utf8mb4_bin：区分大小写
SELECT * FROM test_collation ORDER BY name COLLATE utf8mb4_bin;
-- 结果：A, B, a, b（ASCII码顺序）
</code></pre></p>
<h3>14.3.3 查看排序规则</h3>
<p><strong>查看所有排序规则：</strong></p>
<pre><code class="language-sql">-- 查看utf8mb4的排序规则
SHOW COLLATION WHERE Charset = 'utf8mb4';
<p>-- 查看排序规则变量
SHOW VARIABLES LIKE 'collation%';
</code></pre></p>
<hr>
<h2>14.4 字符集转换</h2>
<h3>14.4.1 转换表字符集</h3>
<p><strong>方法1：ALTER TABLE（推荐）</strong></p>
<pre><code class="language-sql">-- 转换表和所有列的字符集
ALTER TABLE users 
CONVERT TO CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
</code></pre>
<p><strong>方法2：分步转换</strong></p>
<pre><code class="language-sql">-- 1. 修改表默认字符集
ALTER TABLE users 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
<p>-- 2. 修改每列字符集
ALTER TABLE users
MODIFY COLUMN name VARCHAR(100)
CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
</code></pre></p>
<h3>14.4.2 转换数据库字符集</h3>
<p><strong>步骤：</strong></p>
<pre><code class="language-sql">-- 1. 备份数据
mysqldump -u root -p --default-character-set=utf8mb4 mydb &gt; backup.sql
<p>-- 2. 修改数据库字符集
ALTER DATABASE mydb
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;</p>
<p>-- 3. 转换所有表
SELECT CONCAT('ALTER TABLE ', table_name, ' CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;')
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'mydb';</p>
<p>-- 执行生成的SQL
</code></pre></p>
<hr>
<h2>14.5 字符集最佳实践</h2>
<h3>14.5.1 推荐配置</h3>
<p><strong>my.cnf配置：</strong></p>
<pre><code class="language-ini">[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
init_connect = 'SET NAMES utf8mb4'
<p>[client]
default-character-set = utf8mb4</p>
<p>[mysql]
default-character-set = utf8mb4
</code></pre></p>
<p><strong>应用程序连接：</strong></p>
<pre><code class="language-sql">-- 连接后设置字符集
SET NAMES utf8mb4;
<p>-- 或在连接字符串中指定
jdbc:mysql://localhost:3306/mydb?characterEncoding=utf8mb4
</code></pre></p>
<h3>14.5.2 常见问题</h3>
<p><strong>问题1：中文乱码</strong></p>
<pre><code class="language-sql">-- 原因：字符集不一致
-- 解决：统一使用utf8mb4
<p>-- 检查字符集
SHOW VARIABLES LIKE 'character%';</p>
<p>-- 设置字符集
SET NAMES utf8mb4;
</code></pre></p>
<p><strong>问题2：emoji无法存储</strong></p>
<pre><code class="language-sql">-- 原因：使用utf8而不是utf8mb4
-- 解决：转换为utf8mb4
<p>ALTER TABLE users
CONVERT TO CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
</code></pre></p>
<p><strong>问题3：索引长度超限</strong></p>
<pre><code class="language-sql">-- 错误：Specified key was too long; max key length is 767 bytes
-- 原因：utf8mb4每字符4字节，VARCHAR(255)需要1020字节
<p>-- 解决方案1：减少长度
CREATE TABLE users (
email VARCHAR(191) PRIMARY KEY  -- 191*4=764字节
) CHARACTER SET utf8mb4;</p>
<p>-- 解决方案2：使用前缀索引
CREATE TABLE users (
email VARCHAR(255),
INDEX idx_email (email(191))
) CHARACTER SET utf8mb4;</p>
<p>-- 解决方案3：开启innodb_large_prefix（MySQL 5.7+默认开启）
SET GLOBAL innodb_large_prefix = ON;
SET GLOBAL innodb_file_format = Barracuda;
</code></pre></p>

</ul>

<ol>

<li><strong>utf8不支持emoji，utf8mb4支持</strong></li>

</ol>

<ul>
<li>✅ utf8mb4：支持emoji，推荐</li>
<li>❌ utf8：不支持emoji，不推荐</li>
<li>❌ latin1：不支持中文，不推荐</li>
</ul>
<p><strong>排序规则选择：</strong></p>
<ul>
<li>utf8mb4_unicode_ci：推荐（Unicode标准）</li>
