          <div class="vditor-reset post__content"><h1>第16章：备份策略与实践</h1>
<blockquote>
<p>数据备份是DBA最重要的工作之一，没有备份就没有安全感</p>
</blockquote>
<h2>16.1 备份的重要性 ⭐⭐⭐⭐⭐</h2>
<h3>16.1.1 为什么需要备份？</h3>
<p><strong>常见的数据丢失场景：</strong></p>
<ul>
<li>❌ 硬件故障（磁盘损坏、服务器宕机）</li>
<li>❌ 人为误操作（误删数据、DROP TABLE）</li>
<li>❌ 软件Bug（应用程序错误）</li>
<li>❌ 恶意攻击（SQL注入、勒索病毒）</li>
<li>❌ 自然灾害（火灾、地震、水灾）</li>
</ul>
<p><strong>备份的作用：</strong></p>
<ul>
<li>✅ 数据恢复（最重要）</li>
<li>✅ 数据迁移</li>
<li>✅ 搭建从库</li>
<li>✅ 数据分析</li>
<li>✅ 合规要求</li>
</ul>
<p><strong>血泪教训：</strong></p>
<pre><code>没有备份 = 没有工作
没有测试的备份 = 没有备份
</code></pre>
<hr>

<h3>16.2.2 全量备份 vs 增量备份</h3>
<p><strong>全量备份（Full Backup）：</strong></p>
<ul>
<li>备份所有数据</li>
<li>恢复简单</li>
<li>占用空间大</li>
<li>备份时间长</li>
</ul>
<p><strong>增量备份（Incremental Backup）：</strong></p>
<ul>
<li>只备份变化的数据</li>
<li>节省空间</li>
<li>恢复复杂（需要全量+所有增量）</li>
<li>备份时间短</li>
</ul>
<p><strong>差异备份（Differential Backup）：</strong></p>
<ul>
<li>备份自上次全量备份后的所有变化</li>
<li>介于全量和增量之间</li>
<li>恢复需要全量+最后一次差异</li>
</ul>
<h3>16.2.3 热备份 vs 冷备份</h3>
<p><strong>热备份（Hot Backup）：</strong></p>
<ul>
<li>数据库在线备份</li>
<li>不影响业务</li>
<li>推荐使用</li>
</ul>
<p><strong>温备份（Warm Backup）：</strong></p>
<ul>
<li>备份时只能读，不能写</li>
<li>影响部分业务</li>
</ul>
<p><strong>冷备份（Cold Backup）：</strong></p>
<ul>
<li>停止数据库备份</li>
<li>影响业务</li>
<li>不推荐</li>
</ul>
<hr>
<h2>16.3 mysqldump逻辑备份 ⭐⭐⭐⭐⭐</h2>
<h3>16.3.1 mysqldump基本用法</h3>
<pre><code class="language-bash"># 备份单个数据库
mysqldump -u root -p database_name &gt; backup.sql
<h1>备份多个数据库</h1>
<p>mysqldump -u root -p --databases db1 db2 db3 &gt; backup.sql</p>
<h1>备份所有数据库</h1>
<p>mysqldump -u root -p --all-databases &gt; all_backup.sql</p>
<h1>备份单个表</h1>
<p>mysqldump -u root -p database_name table_name &gt; table_backup.sql</p>
<h1>备份多个表</h1>
<p>mysqldump -u root -p database_name table1 table2 &gt; tables_backup.sql
</code></pre></p>
<h3>16.3.2 mysqldump重要参数</h3>
<pre><code class="language-bash"># 完整的备份命令（推荐）
mysqldump -u root -p \
  --single-transaction \      # InnoDB一致性备份（不锁表）
  --master-data=2 \           # 记录binlog位置（注释形式）
  --flush-logs \              # 刷新binlog
  --routines \                # 备份存储过程和函数
  --triggers \                # 备份触发器
  --events \                  # 备份事件
  --hex-blob \                # 二进制数据使用十六进制
  --default-character-set=utf8mb4 \  # 字符集
  database_name &gt; backup.sql
<h1>参数说明：</h1>
<h1>--single-transaction：</h1>
<h1>- 适用于InnoDB</h1>
<h1>- 保证备份的一致性</h1>
<h1>- 不锁表，不影响业务</h1>
<h1>- ⭐ 强烈推荐</h1>
<h1>--master-data=2：</h1>
<h1>- 记录备份时的binlog位置</h1>
<h1>- =1：直接写入CHANGE MASTER语句</h1>
<h1>- =2：以注释形式写入（推荐）</h1>
<h1>- 用于搭建主从复制</h1>
<h1>--flush-logs：</h1>
<h1>- 备份前刷新binlog</h1>
<h1>- 生成新的binlog文件</h1>
<h1>- 便于增量备份</h1>
<h1>--lock-all-tables：</h1>
<h1>- 锁定所有表（MyISAM需要）</h1>
<h1>- 会影响业务</h1>
<h1>- InnoDB不需要</h1>
<h1>--quick：</h1>
<h1>- 快速备份</h1>
<h1>- 不缓冲查询结果</h1>
<h1>- 大表必须使用</h1>
<h1>--compress：</h1>
<h1>- 压缩传输数据</h1>
<h1>- 减少网络传输</h1>
<p></code></pre></p>
<h3>16.3.3 mysqldump备份示例</h3>
<pre><code class="language-bash"># 示例1：备份单个数据库（InnoDB）
mysqldump -u root -p \
  --single-transaction \
  --master-data=2 \
  --flush-logs \
  --routines \
  --triggers \
  mydb &gt; mydb_backup_$(date +%Y%m%d).sql
<h1>示例2：备份所有数据库</h1>
<p>mysqldump -u root -p <br>
--all-databases <br>
--single-transaction <br>
--master-data=2 <br>
--flush-logs <br>
--routines <br>
--triggers <br>
--events &gt; all_backup_$(date +%Y%m%d).sql</p>
<h1>示例3：只备份表结构</h1>
<p>mysqldump -u root -p <br>
--no-data <br>
database_name &gt; schema_only.sql</p>
<h1>示例4：只备份数据</h1>
<p>mysqldump -u root -p <br>
--no-create-info <br>
database_name &gt; data_only.sql</p>
<h1>示例5：备份并压缩</h1>
<p>mysqldump -u root -p <br>
--single-transaction <br>
database_name | gzip &gt; backup.sql.gz</p>
<h1>示例6：远程备份</h1>
<p>mysqldump -h remote_host -u root -p <br>
--single-transaction <br>
database_name &gt; remote_backup.sql
</code></pre></p>
<h3>16.3.4 mysqldump恢复</h3>
<pre><code class="language-bash"># 恢复数据库
mysql -u root -p database_name &lt; backup.sql
<h1>恢复所有数据库</h1>
<p>mysql -u root -p &lt; all_backup.sql</p>
<h1>恢复压缩的备份</h1>
<p>gunzip &lt; backup.sql.gz | mysql -u root -p database_name</p>
<h1>恢复时显示进度</h1>
<p>pv backup.sql | mysql -u root -p database_name</p>
<h1>恢复前创建数据库</h1>
<p>mysql -u root -p -e &quot;CREATE DATABASE IF NOT EXISTS mydb&quot;
mysql -u root -p mydb &lt; backup.sql
</code></pre></p>
<h3>16.3.5 mysqldump的优缺点</h3>
<p><strong>优点：</strong></p>
<ul>
<li>✅ 简单易用</li>
<li>✅ 跨平台、跨版本</li>
<li>✅ 可以查看和编辑备份文件</li>
<li>✅ 可以选择性恢复</li>
<li>✅ 适合小数据库</li>
</ul>
<p><strong>缺点：</strong></p>
<ul>
<li>❌ 备份速度慢</li>
<li>❌ 恢复速度慢</li>
<li>❌ 占用CPU和内存</li>
<li>❌ 不适合大数据库（&gt;100GB）</li>
</ul>
<hr>
<h2>16.4 mysqlpump逻辑备份</h2>
<h3>16.4.1 mysqlpump概述</h3>
<pre><code class="language-bash"># mysqlpump是MySQL 5.7新增的备份工具
# 相比mysqldump的改进：
# ✅ 支持并行备份（多线程）
# ✅ 备份速度更快
# ✅ 支持备份用户
# ✅ 支持压缩
<h1>基本用法</h1>
<p>mysqlpump -u root -p database_name &gt; backup.sql</p>
<h1>并行备份（4个线程）</h1>
<p>mysqlpump -u root -p <br>
--default-parallelism=4 <br>
database_name &gt; backup.sql</p>
<h1>压缩备份</h1>
<p>mysqlpump -u root -p <br>
--compress-output=LZ4 <br>
database_name &gt; backup.sql.lz4
</code></pre></p>

<h2>16.5 XtraBackup物理备份 ⭐⭐⭐⭐⭐</h2>
<h3>16.5.1 XtraBackup概述</h3>
<p><strong>XtraBackup特点：</strong></p>
<ul>
<li>✅ Percona开发的开源工具</li>
<li>✅ 物理备份（直接复制数据文件）</li>
<li>✅ 热备份（不锁表）</li>
<li>✅ 支持增量备份</li>
<li>✅ 备份速度快</li>
<li>✅ 恢复速度快</li>
<li>✅ 适合大数据库</li>
</ul>
<p><strong>版本选择：</strong></p>
<ul>
<li>MySQL 5.7 → XtraBackup 2.4</li>
<li>MySQL 8.0 → XtraBackup 8.0</li>
</ul>
<h3>16.5.2 XtraBackup安装</h3>
<pre><code class="language-bash"># CentOS/RHEL安装
yum install https://repo.percona.com/yum/percona-release-latest.noarch.rpm
yum install percona-xtrabackup-24
<h1>Ubuntu/Debian安装</h1>
<p>wget <a href="https://repo.percona.com/apt/percona-release_latest.generic_all.deb">https://repo.percona.com/apt/percona-release_latest.generic_all.deb</a>
dpkg -i percona-release_latest.generic_all.deb
apt-get update
apt-get install percona-xtrabackup-24</p>
<h1>验证安装</h1>
<p>xtrabackup --version
</code></pre></p>
<h3>16.5.3 XtraBackup全量备份</h3>
<pre><code class="language-bash"># 全量备份
xtrabackup --backup \
  --user=root \
  --password=your_password \
  --target-dir=/backup/full_backup
<h1>参数说明：</h1>
<h1>--backup：执行备份</h1>
<h1>--target-dir：备份目录</h1>
<h1>--user：MySQL用户</h1>
<h1>--password：MySQL密码</h1>
<h1>备份完成后的目录结构：</h1>
<p>/backup/full_backup/
├── ibdata1              # 系统表空间
├── mysql/               # mysql系统库
├── mydb/                # 业务数据库
│   ├── table1.ibd
│   └── table2.ibd

└── xtrabackup_logfile      # 事务日志
</code></pre></p>

--incremental-basedir=/backup/inc1</p>

<h2>16.6 备份策略设计 ⭐⭐⭐⭐⭐</h2>
<h3>16.6.1 备份策略原则</h3>
<p><strong>3-2-1原则：</strong></p>
<ul>
<li><strong>3份数据</strong>：至少保留3份数据副本</li>
<li><strong>2种介质</strong>：使用2种不同的存储介质</li>
<li><strong>1份异地</strong>：至少1份存储在异地</li>
</ul>
<p><strong>备份频率：</strong></p>
<ul>
<li>全量备份：每周1次</li>
<li>增量备份：每天1次</li>
<li>binlog备份：实时或每小时</li>
</ul>
<p><strong>保留策略：</strong></p>
<ul>
<li>每日备份：保留7天</li>
<li>每周备份：保留4周</li>
<li>每月备份：保留12个月</li>
<li>年度备份：永久保留</li>
</ul>
<h3>16.6.2 小型数据库备份策略（&lt;10GB）</h3>
<pre><code class="language-bash"># 策略：每天全量备份
<h1>备份脚本：/backup/scripts/daily_backup.sh</h1>
<p>#!/bin/bash</p>
<p>BACKUP_DIR=&quot;/backup/mysql&quot;
DATE=$(date +%Y%m%d)
MYSQL_USER=&quot;root&quot;
MYSQL_PASSWORD=&quot;your_password&quot;</p>
<h1>创建备份目录</h1>
<p>mkdir -p $BACKUP_DIR/$DATE</p>
<h1>全量备份</h1>
<p>mysqldump -u$MYSQL_USER -p$MYSQL_PASSWORD <br>
--all-databases <br>
--single-transaction <br>
--master-data=2 <br>
--flush-logs <br>
--routines <br>
--triggers <br>
--events | gzip &gt; $BACKUP_DIR/$DATE/all_databases.sql.gz</p>
<h1>删除7天前的备份</h1>
<p>find $BACKUP_DIR -type d -mtime +7 -exec rm -rf {} ;</p>
<h1>定时任务：每天凌晨2点执行</h1>
<h1>crontab -e</h1>
<h1>0 2 * * * /backup/scripts/daily_backup.sh</h1>
<p></code></pre></p>
<h3>16.6.3 中型数据库备份策略（10GB-100GB）</h3>
<pre><code class="language-bash"># 策略：每周全量 + 每天增量
<h1>全量备份脚本：/backup/scripts/weekly_full.sh</h1>
<p>#!/bin/bash</p>

<h1>定时任务：每周日凌晨2点执行</h1>
<h1>0 2 * * 0 /backup/scripts/weekly_full.sh</h1>
<h1>增量备份脚本：/backup/scripts/daily_inc.sh</h1>
<p>#!/bin/bash</p>
<p>BACKUP_DIR=&quot;/backup/mysql&quot;
DATE=$(date +%Y%m%d)</p>
<h1>查找最新的备份（全量或增量）</h1>
<p>LATEST_BACKUP=$(ls -td $BACKUP_DIR/* | head -1)</p>
<h1>增量备份</h1>
<p>xtrabackup --backup <br>
--user=root <br>
--password=your_password <br>
--target-dir=$BACKUP_DIR/inc_$DATE <br>
--incremental-basedir=$LATEST_BACKUP</p>
<h1>定时任务：每天凌晨3点执行（周日除外）</h1>
<h1>0 3 * * 1-6 /backup/scripts/daily_inc.sh</h1>
<p></code></pre></p>
<h3>16.6.4 大型数据库备份策略（&gt;100GB）</h3>
<pre><code class="language-bash"># 策略：XtraBackup + binlog备份
<h1>全量备份：每周1次</h1>
<h1>增量备份：每天1次</h1>
<h1>binlog备份：每小时1次</h1>
<h1>binlog备份脚本：/backup/scripts/binlog_backup.sh</h1>
<p>#!/bin/bash</p>

<p>cd $MYSQL_DATADIR
for file in $(ls mysql-bin.* | head -n -1); do

done</p>

<h1>0 * * * * /backup/scripts/binlog_backup.sh</h1>

<h2>16.7 备份验证与测试 ⭐⭐⭐⭐⭐</h2>
<h3>16.7.1 为什么要验证备份？</h3>

<p><strong>验证内容：</strong></p>
<ul>
<li>✅ 备份文件完整性</li>
<li>✅ 备份可以成功恢复</li>
<li>✅ 恢复后数据正确性</li>
<li>✅ 恢复时间是否满足RTO</li>
</ul>
<h3>16.7.2 备份验证脚本</h3>
<pre><code class="language-bash"># 验证mysqldump备份

TEST_DB=&quot;test_restore&quot;</p>
<h1>1. 检查备份文件是否存在</h1>
<p>if [ ! -f &quot;$BACKUP_FILE&quot; ]; then
echo &quot;备份文件不存在&quot;
exit 1
fi</p>
<h1>2. 检查备份文件大小</h1>
<p>FILE_SIZE=$(stat -f%z &quot;$BACKUP_FILE&quot; 2&gt;/dev/null || stat -c%s &quot;$BACKUP_FILE&quot;)
if [ $FILE_SIZE -lt 1000 ]; then
echo &quot;备份文件太小，可能损坏&quot;
exit 1
fi</p>
<h1>3. 尝试恢复到测试数据库</h1>
<p>mysql -u root -p -e &quot;DROP DATABASE IF EXISTS $TEST_DB&quot;
mysql -u root -p -e &quot;CREATE DATABASE $TEST_DB&quot;
gunzip &lt; $BACKUP_FILE | mysql -u root -p $TEST_DB</p>
<h1>4. 检查恢复结果</h1>
<p>TABLE_COUNT=$(mysql -u root -p -N -e &quot;SELECT COUNT(*) FROM information_schema.TABLES WHERE TABLE_SCHEMA='$TEST_DB'&quot;)
if [ $TABLE_COUNT -eq 0 ]; then
echo &quot;恢复失败，没有表&quot;
exit 1
fi</p>
<p>echo &quot;备份验证成功，共恢复 $TABLE_COUNT 个表&quot;</p>
<h1>5. 清理测试数据库</h1>
<p>mysql -u root -p -e &quot;DROP DATABASE $TEST_DB&quot;
</code></pre></p>
